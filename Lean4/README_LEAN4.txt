1. УСТАНОВКА LEAN 4 (ELAN)
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    РЕКОМЕНДУЕМЫЙ СПОСОБ: ELAN                                │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  elan = Lean version manager (как rustup для Rust)                          │
│  ВСЕГДА используйте elan, не системные пакеты!                              │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

Linux (Ubuntu/Debian):
----------------------
# Шаг 1: Установить elan
curl https://raw.githubusercontent.com/leanprover/elan/master/elan-init.sh -sSf | sh

# Шаг 2: Настроить окружение
source $HOME/.elan/env

# Шаг 3: Проверка
lean --version
lake --version

# Обновление
elan update

# Установка конкретной версии
elan install leanprover/lean4:v4.5.0
elan default leanprover/lean4:v4.5.0

macOS (Homebrew):
-----------------
brew install elan
elan update

Windows:
--------
# PowerShell (Admin):
curl https://raw.githubusercontent.com/leanprover/elan/master/elan-init.sh -sSf | sh

# Или скачать с https://leanprover.github.io/

Docker (универсально):
----------------------
docker run -it --rm leanprover/lean4 lean --version

Установка VS Code + Lean 4:
---------------------------
1. Extensions (Ctrl+Shift+X)
2. Установить: "Lean 4" от leanprover
3. Установить: "Lean 4 InfoView" (входит в основное)
4. Настроить путь к lean если нужно

Настройки VS Code (settings.json):
----------------------------------
{
    "lean4.serverPath": "lean",
    "lean4.infoViewAllGoal": true,
    "lean4.infoViewShowExpectedType": true
}

3. БЫСТРЫЙ СТАРТ
================================================================================

# Запуск REPL
lean

# Вы увидите:
# Lean (version 4.5.0, ...)
# #

# Первый код:
# #eval 2 + 2
# 4

# #check Nat
# Nat : Type

# Выход:
# #exit

# Создание проекта:
lake new my_project
cd my_project
lake build

# Структура проекта:
my_project/
├── lakefile.toml      # Конфигурация проекта
├── lean-toolchain     # Версия Lean
├── MyProject/
│   └── Main.lean      # Исходный код
└── lake-manifest.json # Зависимости

# Запуск файла:
lean --run Main.lean

# Компиляция:
lake build
lake exe my_project

4. УПРАВЛЕНИЕ ПАКЕТАМИ (LAKE)
================================================================================

# Lake = Lean package manager (как cargo)

# Создание проекта
lake new my_project

# Добавление зависимости
# В lakefile.toml:
[[require]]
name = "mathlib"
git = "https://github.com/leanprover-community/mathlib4.git"

# Установка зависимостей
lake update
lake build

# Основные команды:
lake build          # Собрать проект
lake test           # Запустить тесты
lake clean          # Очистить
lake update         # Обновить зависимости
lake exe name       # Запустить исполняемый файл
lake run            # Собрать и запустить

# Mathlib (математическая библиотека):
# Это ОСНОВНАЯ библиотека для Lean!
# 100,000+ теорем и определений

5. ПОДВОДНЫЕ КАМНИ УСТАНОВКИ
================================================================================

⚠️ Проблема 1: elan не установлен
   lean: command not found
   
   Решение: 
   curl https://raw.githubusercontent.com/leanprover/elan/master/elan-init.sh -sSf | sh

⚠️ Проблема 2: VS Code extension не работает
   Lean сервер не запускается
   
   Решение:
   - Проверить что lean в PATH
   - Перезапустить VS Code
   - Проверить Output → Lean 4

⚠️ Проблема 3: Mathlib не загружается
   Cannot find mathlib
   
   Решение:
   lake update
   lake build

⚠️ Проблема 4: Долгая первая компиляция
   Первый lake build может занять 30-60 минут
   
   Решение: Это нормально - компиляция Mathlib
   Используйте кэш: ~/.cache/lean

⚠️ Проблема 5: Версия Lean не совпадает
   Version mismatch error
   
   Решение:
   elan override set leanprover/lean4:v4.5.0

⚠️ Проблема 6: InfoView не показывает цели
   Proof goals не видны
   
   Решение:
   - Включить Lean 4 InfoView в VS Code
   - Нажать на строку с sorry
   - Проверить настройки infoViewAllGoal
