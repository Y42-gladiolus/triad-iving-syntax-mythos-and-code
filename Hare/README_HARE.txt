3. БЫСТРЫЙ СТАРТ
================================================================================

# Hare REPL (нет REPL, компиляция только)
# Hare компилируется напрямую

# Создание проекта
mkdir my_project && cd my_project

# Структура проекта:
my_project/
├── main.ha        # Главный файл
└── README.md      # Документация

# Простая программа (main.ha)
use fmt;

export fn main() void = {
  fmt::printline("Hello, Hare!");
};

# Компиляция
hare build

# Запуск
./my_project

# Compile and run
hare run main.ha

5. ПОДВОДНЫЕ КАМНИ УСТАНОВКИ
================================================================================

⚠️ Проблема 1: Очень новый язык
   Hare в активной разработке (alpha)
   
   Решение:
   Следить за обновлениями
   Использовать latest commit

⚠️ Проблема 2: Меньше библиотек
   Экосистема только формируется
   
   Решение:
   Использовать POSIX APIs
   Писать свои библиотеки

⚠️ Проблема 3: Документация
   Документация ещё развивается
   
   Решение:
   Читать source code
   Присоединиться к IRC/Matrix

⚠️ Проблема 4: IDE поддержка
   Ограниченная поддержка
   
   Решение:
   Использовать Vim/Emacs
   Или текстовый редактор + CLI

⚠️ Проблема 5: Нет Windows support
   Только POSIX системы
   
   Решение:
   Использовать Linux/BSD
   Или WSL

⚠️ Проблема 6: Нет package manager
   Manual dependency management
   
   Решение:
   Clone repositories manually
   Follow project updates

echo "alias harerun='hare run'" >> ~/.zshrc

echo "alias harebuild='hare build'" >> ~/.bashrc
echo "alias harebuild='hare build'" >> ~/.zshrc

# Применить изменения
source ~/.bashrc
source ~/.zshrc
source ~/.mkshrc 2>/dev/null || true
