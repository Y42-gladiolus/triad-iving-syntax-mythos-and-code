3. ОПТИМИЗАЦИЯ: КАК УСКОРИТЬ JAVASCRIPT
================================================================================

Совет 1: Используйте async/await для I/O
----------------------------------------
// МЕДЛЕННО (последовательно):
const data1 = await fetch(url1);
const data2 = await fetch(url2);
const data3 = await fetch(url3);

// БЫСТРО (параллельно):
const [data1, data2, data3] = await Promise.all([
  fetch(url1),
  fetch(url2),
  fetch(url3)
]);

// ⚡ Ускорение: 3x для независимых запросов!

Совет 2: Избегайте мутаций
--------------------------
// МЕДЛЕННО (мутация):
arr.push(item);

// БЫСТРО (immutable):
const newArr = [...arr, item];

// ⚡ V8 оптимизирует лучше!

Совет 3: Используйте typed arrays для чисел
-------------------------------------------
// МЕДЛЕННО (обычный массив):
const arr = [1, 2, 3, 4, 5];

// БЫСТРО (typed array):
const arr = new Int32Array([1, 2, 3, 4, 5]);

// ⚡ Ускорение: 2-5x для численных операций!

Совет 4: Избегайте eval и new Function
--------------------------------------
// МЕДЛЕННО (eval):
const result = eval("2 + 2");

// БЫСТРО (прямое вычисление):
const result = 2 + 2;

// ⚡ eval отключает оптимизации V8!

Совет 5: Используйте Web Workers для CPU-bound
----------------------------------------------
// МЕДЛЕННО (блокирует UI):
const result = heavyComputation();

// БЫСТРО (отдельный поток):
const worker = new Worker('worker.js');
worker.postMessage(data);

// ⚡ UI не блокируется!

9. БЕНЧМАРК: ПРОВЕРЬТЕ САМИ
================================================================================

Создайте benchmark.js:
----------------------
function sumEnum(n) {
  let sum = 0;
  for (let i = 1; i <= n; i++) {
    sum += i;
  }
  return sum;
}

const start = performance.now();
const result = sumEnum(1_000_000_000);
const end = performance.now();

console.log(`Sum: ${result}`);
console.log(`Time: ${end - start} ms`);

Запуск:
-------
node benchmark.js
