3. ОПТИМИЗАЦИЯ: КАК УСКОРИТЬ FORTH
================================================================================

Совет 1: Используйте inline слова
---------------------------------
\ МЕДЛЕННО (вызов слова):
: add ( a b -- c )
  +
;

\ БЫСТРО (inline):
: add ( a b -- c )
  +
; inline

\ ⚡ Устраняет overhead вызова!

Совет 2: Минимизируйте stack манипуляции
----------------------------------------
\ МЕДЛЕННО (лишние операции):
: add ( a b -- c )
  swap swap +
;

\ БЫСТРО (прямая операция):
: add ( a b -- c )
  +
;

\ ⚡ Меньше stack операций = быстрее!

Совет 3: Используйте register variables (Gforth)
------------------------------------------------
\ МЕДЛЕННО (память):
variable count

\ БЫСТРО (регистр):
: example
  { a b c -- result }
  a b + c *
;

\ ⚡ Регистры быстрее памяти!

Совет 4: Компилируйте в native code
-----------------------------------
\ МЕДЛЕННО (интерпретация):
gforth script.fs

\ БЫСТРО (компиляция):
gforth script.fs -e "compile bye"
./script

\ ⚡ Native code быстрее интерпретации!

Совет 5: Избегайте рекурсии в hot path
--------------------------------------
\ МЕДЛЕННО (рекурсия):
: factorial ( n -- n! )
  dup 1 <= if
    drop 1
  else
    dup 1 - factorial *
  then
;

\ БЫСТРО (итерация):
: factorial ( n -- n! )
  1 swap 1 + 1 do
    i *
  loop
;

\ ⚡ Итерация быстрее рекурсии!

Forth + C (через FFI):
----------------------
\ Вызов C функции
external: printf ( addr -- )

\ Интеграция в C код
\ forth_eval("2 2 + .");

Forth + Assembly:
-----------------
\ Вставка assembly кода
code fast-add ( a b -- c )
  pop ax
  pop bx
  add ax, bx
  push ax
  next
end-code

Forth + Python (через subprocess):
----------------------------------
\ Вызов Python скрипта
s" python script.py" system

Forth + MiniZinc (через CLI):
-----------------------------

----------------------

  0 swap 1 + 1 do
    i +
  loop
;

utime drop constant start
100000000 sum-enum
utime drop start - constant elapsed

." Sum computed in " elapsed . ." microseconds" cr

Запуск:
-------
gforth benchmark.fs
