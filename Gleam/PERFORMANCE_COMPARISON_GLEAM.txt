3. ОПТИМИЗАЦИЯ: КАК УСКОРИТЬ GLEAM
================================================================================

Совет 1: Используйте BEAM target для backend
--------------------------------------------
// МЕДЛЕННО (JavaScript target для backend):
gleam build --target javascript

// БЫСТРО (Erlang target для backend):
gleam build --target erlang

// ⚡ Ускорение: 2-5x для backend задач!

Совет 2: Используйте pattern matching вместо if/else
----------------------------------------------------
// МЕДЛЕННО (вложенные if):
if condition {
  if other {
    result1
  } else {
    result2
  }
} else {
  result3
}

// БЫСТРО (pattern matching):
case #(condition, other) {
  #(True, True) -> result1
  #(True, False) -> result2
  #(False, _) -> result3
}

// ⚡ Pattern matching оптимизирован в BEAM!

Совет 3: Используйте tail-рекурсию
----------------------------------
// МЕДЛЕННО (обычная рекурсия):
pub fn sum(list) {
  case list {
    [] -> 0
    [head, ..tail] -> head + sum(tail)
  }
}

// БЫСТРО (tail-рекурсия):
pub fn sum(list) {
  fn loop(acc, list) {
    case list {
      [] -> acc
      [head, ..tail] -> loop(acc + head, tail)
    }
  }(0, list)
}

// ⚡ BEAM оптимизирует tail-рекурсию!

Совет 4: Используйте Result.try для цепочек
-------------------------------------------
// МЕДЛЕННО (вложенные case):
case operation1() {
  Ok(a) -> case operation2(a) {
    Ok(b) -> Ok(b)
    Error(e) -> Error(e)
  }
  Error(e) -> Error(e)
}

// БЫСТРО (use синтаксис):
use a <- result.try(operation1())
use b <- result.try(operation2(a))
Ok(b)

// ⚡ Чище и меньше ошибок!

9. БЕНЧМАРК: ПРОВЕРЬТЕ САМИ
================================================================================

Создайте benchmark.gleam:
-------------------------
import gleam/io
import gleam/int
import gleam/list

pub fn sum_enum(n: Int) -> Int {
  list.range(1, n)
  |> list.sum
}

pub fn main() {
  let start = io.now()
  let result = sum_enum(1_000_000)
  let end = io.now()
  io.println("Sum: " <> int.to_string(result))
  io.println("Time: " <> int.to_string(end - start) <> " μs")
}

Запуск:
-------
gleam run

Сравнение с Elixir:
-------------------
# Gleam будет немного медленнее (компиляция)
# Но меньше runtime ошибок!
