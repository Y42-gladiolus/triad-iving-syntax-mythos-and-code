** Правила чтения (алгоритм)
:PROPERTIES:
:CUSTOM_ID: reading-algorithm
:END:

#+BEGIN_SRC text
FUNCTION read_concept(concept_name):
    ref  = open(f"01_{concept_name}-code.org")   # Тело
    fable = open(f"{concept_name}-fable.org")    # Душа
    praxis = open(f"03_{concept_name}/main.*")   # Действие

    PARALLEL:
        READ ref   → extract invariants
        READ fable → extract meaning
        READ praxis → extract behavior

    RETURN synthesize(ref, fable, praxis)
END
#+END_SRC

*Почему параллельно?*
- Слой *Reference* без *Fable* → сухая спецификация без смысла
- Слой *Fable* без *Praxis* → поэзия без проверки реальностью
- Слой *Praxis* без *Reference* → код без гарантий

** Почему префиксы 00_, 01_, 02_, 03_, 99_?
:PROPERTIES:
:CUSTOM_ID: prefix-logic
:END:

| Префикс | Семантика | Сортировка |
|---------+-----------+------------|
| `00_` | Фундамент, манифесты, общие правила | В самом верху |
| `01_` | Справочный слой (Reference) | После фундамента |
| `02_` | Философский слой (Fable) | После справочника |
| `03_` | Практический слой (Praxis) | После философии |
| `99_` | Декоративный слой (Easter Egg) | В самом низу, как вишенка |

*Принцип:* файловая система как навигация по смыслу.

** Как добавлять новый язык/концепт?
:PROPERTIES:
:CUSTOM_ID: adding-new
:END:

1. Создайте папку языка: `MyLang/` (если ещё нет)
2. Добавьте в три слоя:
   - `01_MyLang-code.org` — синтаксис, типы, инварианты
   - `MyLang-fable.org` — почему этот язык, метафоры, история
   - `03_MyLang/main.ext` — рабочий пример + тест
3. Обновите `CLASSIFICATION.md` (если язык новый в списке)
4. *Опционально:* `99_MyLang-illuminati.org` — если есть эстетический слой
