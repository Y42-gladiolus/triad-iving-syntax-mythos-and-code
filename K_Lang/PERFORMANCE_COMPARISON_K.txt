3. ОПТИМИЗАЦИЯ: КАК УСКОРИТЬ K
================================================================================

Совет 1: Используйте векторные операции вместо циклов
-----------------------------------------------------
/ МЕДЛЕННО (цикл):
sum: 0
do[1000000; sum+: x]

/ БЫСТРО (вектор):
+/ x

/ ⚡ Ускорение: 100-1000x!

Совет 2: Используйте встроенные агрегаты
----------------------------------------
/ МЕДЛЕННО (ручная сумма):
{[x] s:0; do[#x; s+: x[y]]} x

/ БЫСТРО (встроенная):
+/ x

/ ⚡ Ускорение: 10-100x!

Совет 3: Используйте kdb+ для временных рядов
---------------------------------------------
/ МЕДЛЕННО (Python pandas):
# 10-30 секунд для 10M записей

/ БЫСТРО (kdb+):
/ 0.1-1 секунда для 10M записей

/ ⚡ Ускорение: 10-30x!

Совет 4: Избегайте скалярных операций
-------------------------------------
/ МЕДЛЕННО (скаляр):
{[x] x[0] + x[1] + x[2] + ...} x

/ БЫСТРО (вектор):
+/ x

/ ⚡ Ускорение: 10-100x!

Совет 5: Используйте in-memory базу данных
------------------------------------------
/ МЕДЛЕННО (диск):
read CSV, process, write

/ БЫСТРО (kdb+ in-memory):
Загрузить в память, обрабатывать

/ ⚡ Ускорение: 100-1000x для частых запросов!

---------------------------
import qpython
q = QConnection('localhost', 5000)

K + R (через Rkdb):
-------------------
library(Rkdb)
con <- dbConnect(kdb(), "localhost", 5000)
dbGetQuery(con, "select from trade")

K + Julia (через Kdb.jl):
-------------------------
using Kdb
conn = Connection("localhost", 5000)
query(conn, "select from trade")

-------------------------
runMinizinc: { [model; data]
  system "minizinc --solver CBC ", model, " ", data

9. БЕНЧМАРК: ПРОВЕРЬТЕ САМИ
================================================================================

Создайте benchmark.k:
---------------------
/ Сумма квадратов
sumSquares: { +/ x * x }

/ Тест
n: 1000000
x: 1 + ! n

/ Замер времени
\t sumSquares x

Запуск:
-------
k benchmark.k

Сравнение с Python:
-------------------
# Python
import time
n = 1000000
x = list(range(1, n+1))
start = time.time()
sum(i*i for i in x)
print(time.time() - start)

# K будет в 100-1000 раз быстрее!
