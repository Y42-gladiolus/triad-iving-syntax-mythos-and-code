Linux (Ubuntu/Debian):
----------------------
# Вариант 1: Через apt (может быть старая версия)
sudo apt update
sudo apt install pil

# Вариант 2: Из исходников (рекомендуется!)
git clone https://github.com/picolisp/picolisp.git
cd picolisp/src
make
sudo make install

# Проверка
pil --version
# или
which pil

macOS (Homebrew):
-----------------
brew install picolisp

# Проверка
pil --version

Windows:
--------
# Требуется WSL или Cygwin
# Скачать исходники с GitHub
# Скомпилировать через make

# Проверка в PowerShell:
pil --version

Docker (универсально):
----------------------
docker run -it --rm bengoeldi/picolisp pil

----------------------------
# Встроен в Emacs
(add-to-list 'auto-mode-alist '("\\.l$" . lisp-mode))

3. БЫСТРЫЙ СТАРТ
================================================================================

# PicoLisp REPL
pil

# В REPL:
: (+ 2 2)
-> 4
: (bye)  # Выход

# Запуск скрипта
pil script.l

# Запуск с кодом из командной строки
pil -e '(println (+ 2 2))'

# Компиляция в bytecode
pil -c script.l

# Создание проекта
mkdir my_project && cd my_project

# Структура проекта:
my_project/
├── main.l         # Главный файл
├── lib/           # Библиотеки
│   └── utils.l
├── db/            # База данных
└── www/           # Веб файлы

5. ПОДВОДНЫЕ КАМНИ УСТАНОВКИ
================================================================================

⚠️ Проблема 1: Символы как ячейки памяти
   Непривычно для Lisp-еров
   
   Решение:
   Читать документацию о символах
   Практиковаться ежедневно

⚠️ Проблема 2: Нет традиционных списков
   Символы вместо cons-ячеек
   
   Решение:
   Изучить модель памяти PicoLisp
   Читать "The PicoLisp Reference"

⚠️ Проблема 3: Встроенная база данных
   Непривычно для Lisp
   
   Решение:
   Изучить Pilog
   Читать примеры использования

⚠️ Проблема 4: Мало документации
   Меньше чем Common Lisp
   
   Решение:
   Читать официальную документацию
   Изучать примеры на GitHub

⚠️ Проблема 5: Синтаксис отличается
   От традиционного Lisp
   
   Решение:
   Практиковаться с REPL
   Использовать vim pil.vim

⚠️ Проблема 6: Coroutines (co)
   Непривычно для Lisp-еров
   
   Решение:
   Изучить документацию по co
   Практиковаться с примерами

echo "alias pl='pil'" >> ~/.zshrc

echo "alias plc='pil -c'" >> ~/.bashrc
echo "alias plc='pil -c'" >> ~/.zshrc

echo "alias ple='pil -e'" >> ~/.bashrc
echo "alias ple='pil -e'" >> ~/.zshrc

# Применить изменения
source ~/.bashrc
source ~/.zshrc
source ~/.mkshrc 2>/dev/null || true
