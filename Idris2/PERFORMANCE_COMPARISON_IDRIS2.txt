┌──────────────────────────────────────────────────────────────────────────────┐
│                    РЕАЛЬНАЯ ПРОИЗВОДИТЕЛЬНОСТЬ IDRIS2                        │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  МИФ: "Зависимые типы замедляют выполнение"                                 │
│  РЕАЛЬНОСТЬ: Зависимые типы проверяются на compile-time, не runtime!        │
│                                                                              │
│  ⚡ Где Idris2 БЫСТРЫЙ:                                                      │
│  • Compile-time проверки (не runtime оверхед)                               │
│  • Оптимизированный код (через Scheme/GHC backend)                          │
│  • Доказательства стираются при компиляции                                  │
│  • Correctness by construction (меньше багов = меньше отладки)              │
│                                                                              │
│  🐌 Где Idris2 МЕДЛЕННЫЙ:                                                    │
│  • Время компиляции (проверка типов сложная)                                │
│  • Численные вычисления (медленнее Julia/Rust/C)                            │
│  • Параллельные вычисления (ограниченная поддержка)                         │
│  • Runtime если доказательства не стираются                                 │
│                                                                              │
│  ⚡ Erasure: доказательства удаляются при компиляции!                       │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

3. ОПТИМИЗАЦИЯ: КАК УСКОРИТЬ IDRIS2
================================================================================

Совет 1: Используйте %erase для доказательств
---------------------------------------------
-- Доказательства стираются при компиляции
total
safeDivide : (x : Integer) -> (y : Integer) -> 
             {auto prf : y /= 0} -> Integer
safeDivide x y {prf} = x `div` y

-- prf будет стёрт в runtime!

Совет 2: Используйте Int не Integer для чисел
---------------------------------------------
-- МЕДЛЕННО (Integer - произвольная точность):
sum : List Integer -> Integer
sum = foldl (+) 0

-- БЫСТРО (Int - машинные целые):
sum : List Int -> Int
sum = foldl (+) 0

-- ⚡ Ускорение: 5-10x для интенсивных вычислений!

Совет 3: Избегайте доказательств в горячем коде
-----------------------------------------------
-- МЕДЛЕННО (доказательство в runtime):
process : (xs : Vect n a) -> (prf : n > 0) -> a
process xs prf = head xs

-- БЫСТРО (доказательство в типе):
process : (xs : Vect (S n) a) -> a
process xs = head xs

-- ⚡ prf проверяется на compile-time, не runtime!

Совет 4: Используйте --compile-to-cheap для разработки
------------------------------------------------------
# МЕДЛЕННО (полная компиляция):
idris2 --build test.ipkg

# БЫСТРО (для разработки):
idris2 --compile-to-cheap test.idr

-- ⚡ Ускорение компиляции: 5-10x!

Совет 5: Inline критические функции
-----------------------------------
-- Принудительная inline оптимизация
%inline
fastAdd : Int -> Int -> Int
fastAdd x y = x + y

-- ⚡ Ускорение: 2-3x для маленьких функций!

9. БЕНЧМАРК: ПРОВЕРЬТЕ САМИ
================================================================================

Создайте Benchmark.idr:
-----------------------
module Main

import System

-- Численный бенчмарк
sumTo : Int -> Int
sumTo n = go 0 0
  where
    go : Int -> Int -> Int
    go acc Z = acc
    go acc (S k) = go (acc + cast k) k

-- Векторный бенчмарк
vecSum : Vect n Int -> Int
vecSum [] = 0
vecSum (x :: xs) = x + vecSum xs

main : IO ()
main = do
  start <- time Monotonic
  let result = sumTo 1000000
  end <- time Monotonic
  putStrLn $ "Sum: " ++ show result
  putStrLn $ "Time: " ++ show (diffTime end start)

Компиляция и запуск:
--------------------
idris2 --build benchmark.ipkg
./build/exec/benchmark

Сравнение с доказательствами vs без:
------------------------------------
-- С доказательствами (медленнее компиляция, быстрее runtime)
safeHead : (xs : Vect (S n) a) -> a
safeHead (x :: xs) = x

-- Без доказательств (быстрее компиляция, runtime проверки)
unsafeHead : List a -> a
unsafeHead (x :: xs) = x
unsafeHead [] = crash "empty list"
