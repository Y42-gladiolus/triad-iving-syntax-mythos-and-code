2. C++ ИНТЕРОПЕРАБЕЛЬНОСТЬ
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    CARBON C++ INTEROPERABILITY                               │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  BIDIRECTIONAL INTEROP:                                                      │
│  • Вызов C++ кода из Carbon без overhead                                    │
│  • Вызов Carbon кода из C++ без overhead                                    │
│  • Нет bindings/wrappers нужно                                              │
│  • Прямой импорт C++ библиотек                                              │
│                                                                              │
│  GRADUAL MIGRATION:                                                          │
│  • Постепенная миграция с C++ на Carbon                                     │
│  • Можно мигрировать файл за файлом                                         │
│  • Нет need to rewrite everything at once                                   │
│  • C++ и Carbon в одном проекте                                             │
│                                                                              │
│  LIBRARY COMPATIBILITY:                                                      │
│  • STL библиотеки работают напрямую                                         │
│  • Boost библиотеки работают напрямую                                       │
│  • C++ ABI совместимость                                                    │
│  • Нет FFI overhead                                                         │
│                                                                              │
│  EXAMPLES:                                                                   │
│  • import C++ library "iostream"                                            │
│  • std.cout << "Hello"                                                      │
│  • Carbon code вызывается из C++ как обычный C++                            │
│                                                                              │
│  ADVANTAGES vs OTHER LANGUAGES:                                              │
│  • Rust: C++ interop требует bindings                                       │
│  • D: C++ interop есть но сложнее                                           │
│  • Carbon: Лучшая C++ interop в индустрии                                   │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

3. СИСТЕМА ТИПОВ
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    CARBON TYPE SYSTEM                                        │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  STATIC TYPING:                                                              │
│  • Все типы известны на compile-time                                        │
│  • Type inference (вывод типов)                                             │
│  • No type annotations needed most of the time                              │
│  • Strong type safety                                                       │
│                                                                              │
│  GENERICS (вместо C++ templates):                                            │
│  • Проще чем C++ templates                                                  │
│  • Нет template metaprogramming complexity                                  │
│  • Better error messages                                                    │
│  • Faster compilation                                                       │
│                                                                              │
│  EXAMPLE:                                                                    │
│  • fn Max[T: Comparable](a: T, b: T) -> T                                   │
│  • vs C++: template<typename T> T Max(T a, T b)                             │
│                                                                              │
│  TYPE INFERENCE:                                                             │
│  • var x = 5  # выводит Int                                                 │
│  • let y: Int = 10  # explicit                                              │
│  • Works across functions                                                   │
│  • Fast inference                                                           │
│                                                                              │
│  NO UNDEFINED BEHAVIOR (цель):                                               │
│  • Нет undefined behavior (цель дизайна)                                    │
│  • Все поведение определено                                                 │
│  • Безопаснее чем C++                                                       │
│  • Predictable semantics                                                    │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

│                    CARBON ERROR HANDLING                                     │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  RESULT TYPES (планируется):                                                 │
│  • Result<T, E> для ошибок                                                  │

│  • Можно использовать exceptions                                            │
│  • C++ exceptions совместимость                                             │
│  • Опционально (можно отключить)                                            │
│  • Для C++ interop                                                          │
│                                                                              │
│  ERROR MESSAGES:                                                             │
│  • Отличные сообщения об ошибках                                            │
│  • Цветной вывод                                                            │
│  • Предложения по исправлению                                               │
│  • Better than C++                                                          │
│                                                                              │
│  ASSERTIONS:                                                                 │
│  • Assert для проверок                                                      │
│  • Compile-time assertions                                                  │
│  • Runtime assertions                                                       │
│  • Debug builds                                                             │
│                                                                              │
│  EXAMPLE:                                                                    │
│  • fn Divide(a: i32, b: i32) -> Result<i32, Error>                          │
│  • if b == 0: return Error.DivisionByZero                                   │
│  • return Ok(a / b)                                                         │
│                                                                              │

7. MEMORY SAFETY
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    CARBON MEMORY SAFETY                                      │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  SAFETY GOALS:                                                               │
│  • Memory safety без GC (цель)                                              │
│  • Нет undefined behavior (цель)                                            │
│  • Нет buffer overflows (цель)                                              │
│  • Нет use-after-free (цель)                                                │
│                                                                              │
│  CURRENT STATUS (2024):                                                      │
│  • В активной разработке                                                    │
│  • Нет production ready memory safety                                       │
│  • Basic safety features есть                                               │
│  • Full safety в разработке                                                 │
│                                                                              │
│  SAFE BY DEFAULT:                                                            │
│  • Безопасные указатели по умолчанию                                        │
│  • Bounds checking                                                          │
│  • Null pointer checks                                                      │
│  • Type safety                                                              │
│                                                                              │
│  UNSAFE ESCAPE HATCHES:                                                      │
│  • Можно использовать unsafe когда нужно                                    │
│  • Явно помечается                                                          │
│  • Для performance-critical кода                                            │
│  • Для C++ interop                                                          │
│                                                                              │
│  COMPARISON:                                                                 │
│  • C++: Нет safety by default                                               │
│  • Rust: Safety by default (готово)                                         │
│  • Carbon: Safety by default (в разработке)                                 │
│  • D: Safety optional                                                       │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
