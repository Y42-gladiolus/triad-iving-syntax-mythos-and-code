1. АРХИТЕКТУРА И ДИЗАЙН
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    WREN ARCHITECTURE                                         │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  IMPLEMENTATION:                                                             │
│  • Написан на C (для встраиваемости и переносимости)                        │
│  • Байт-код интерпретатор                                                   │
│  • Нет JIT (для простоты)                                                   │
│  • Кроссплатформенный (Linux, macOS, Windows)                               │
│                                                                              │
│  DESIGN PHILOSOPHY:                                                          │
│  • Small (маленький, легко встроить)                                        │
│  • Fast (быстрый для скриптового языка)                                     │
│  • Simple (простой синтаксис)                                               │
│  • Class-based (не prototype-based как JavaScript)                          │
│  • Embeddable (легко встроить в приложения)                                 │
│  • No GC pauses (предсказуемая производительность)                          │
│                                                                              │
│  KEY FEATURES:                                                               │
│  • Class-based OOP                                                          │
│  • Fibers (легковесные потоки)                                              │
│  • Closures                                                                 │
│  • Dynamic typing                                                           │
│  • Automatic memory management                                              │
│  • Small footprint (~50 KB)                                                 │
│  • Fast startup                                                             │
│                                                                              │
│  STATUS (2024):                                                              │
│  • Stable release                                                           │
│  • Активная разработка                                                      │
│  • Production ready                                                         │
│  • Community-driven                                                         │
│  • Open source (MIT license)                                                │
│  • Created by Bob Nystrom (автор "Crafting Interpreters")                   │
│                                                                              │
│  COMPARISON:                                                                 │
│  • vs Lua: Class-based (не prototype), проще API                            │
│  • vs Python: Меньше, быстрее, проще встроить                               │
│  • vs JavaScript: Class-based, нет prototype confusion                      │
│  • vs Ruby: Меньше, быстрее, проще                                          │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

2. СИСТЕМА ТИПОВ
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    WREN TYPE SYSTEM                                          │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  DYNAMIC TYPING:                                                             │
│  • Типы проверяются на runtime                                              │
│  • Нет type annotations                                                     │
│  • Flexible и гибкий                                                        │

│                                                                              │

│  • Num (numbers, double precision)                                          │
│  • String (UTF-8 strings)                                                   │
│  • Bool (true, false)                                                       │
│  • Null (null value)                                                        │
│  • Class (class instances)                                                  │
│  • List (dynamic arrays)                                                    │
│  • Map (hash maps)                                                          │
│  • Fiber (coroutines)                                                       │

│                                                                              │
│  NO STATIC TYPES:                                                            │
│  • Нет compile-time type checking                                           │
│  • Runtime type errors возможны                                             │
│  • Testing important                                                        │
│  • Faster development                                                       │
│                                                                              │
│  TYPE OPERATIONS:                                                            │
│  • is operator (type checking)                                              │
│  • toString() (string conversion)                                           │
│  • Dynamic dispatch                                                         │
│  • Duck typing                                                              │
│                                                                              │
│  FOREIGN TYPES:                                                              │
│  • C pointers через Foreign class                                           │
│  • Embedding API для C integration                                          │
│  • Custom types через C API                                                 │
│  • Seamless C integration                                                   │
│                                                                              │

│                                                                              │

│  • Mark-and-sweep collector                                                 │
│  • Small GC pauses                                                          │
│  • Tunable GC parameters                                                    │
│  • Low memory overhead                                                      │
│                                                                              │
│  FOREIGN MEMORY:                                                             │
│  • Foreign class для C pointers                                             │
│  • Manual memory management для Foreign                                     │
│  • Finalizers для cleanup                                                   │
│  • C integration с memory safety                                            │
│                                                                              │
│  NO REFERENCE COUNTING:                                                      │
│  • Нет reference counting (как Python)                                      │
│  • Нет circular reference problems                                          │
│  • Simpler than reference counting                                          │
│  • Better performance                                                       │

│  MEMORY FOOTPRINT:                                                           │

│  • Low memory usage                                                         │

│  • Predictable memory usage                                                 │

6. CONCURRENCY MODEL
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    WREN CONCURRENCY                                          │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  FIBERS (COROUTINES):                                                        │
│  • Легковесные потоки                                                       │
│  • Cooperative multitasking                                                 │
│  • No threads (по умолчанию)                                                │
│  • Like Lua coroutines                                                      │
│                                                                              │
│  FIBER OPERATIONS:                                                           │
│  • Fiber.new() для создания                                                 │
│  • Fiber.call() для запуска                                                 │
│  • Fiber.yield() для pause                                                  │
│  • Fiber.try() для error handling                                           │
│                                                                              │
│  NO THREADS:                                                                 │
│  • Нет native threads                                                       │
│  • Single-threaded по умолчанию                                             │
│  • Simpler concurrency model                                                │
│  • No race conditions                                                       │
│                                                                              │
│  EMBEDDING THREADS:                                                          │
│  • Можно запускать в multiple threads                                       │
│  • Один VM per thread                                                       │
│  • No shared state между VMs                                                │
│  • Thread-safe с proper isolation                                           │
│                                                                              │
│  ASYNC PROGRAMMING:                                                          │
│  • Нет built-in async/await                                                 │
│  • Fibers для cooperative multitasking                                      │
│  • Event loop через C API                                                   │
│  • Custom async implementations                                             │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
