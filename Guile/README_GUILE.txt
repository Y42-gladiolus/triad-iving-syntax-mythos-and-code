Linux (Ubuntu/Debian):
----------------------
# Вариант 1: Через apt
sudo apt update
sudo apt install guile-3.0 guile-3.0-dev

# Вариант 2: Из исходников (рекомендуется!)
wget https://ftp.gnu.org/gnu/guile/guile-3.0.9.tar.gz
tar xzf guile-3.0.9.tar.gz
cd guile-3.0.9
./configure
make
sudo make install

# Проверка
guile --version

macOS (Homebrew):
-----------------
brew install guile

# Проверка
guile --version

Windows:
--------
# Требуется WSL или Cygwin

# Скомпилировать через Cygwin

# Проверка в PowerShell:

Docker (универсально):
----------------------
docker run -it --rm guile guile

3. БЫСТРЫЙ СТАРТ
================================================================================

# Guile REPL
guile

# В REPL:
scheme@(guile-user)> (+ 2 2)
$1 = 4
scheme@(guile-user)> (exit)  # Выход

# Запуск скрипта
guile script.scm

# Запуск с кодом из командной строки
guile -c '(+ 2 2)'

# Запуск через shebang
#!/usr/bin/env guile
(display "Hello, Guile!")

# Компиляция в bytecode
guild compile script.scm

# Создание проекта
mkdir my_project && cd my_project

# Структура проекта:

├── main.scm       # Главный файл

4. УПРАВЛЕНИЕ ПАКЕТАМИ
================================================================================

# Guile имеет пакетный менеджер (Guild)

# Поиск пакетов
guild search package_name

# Установка пакета
guild install package_name

# Обновление пакетов
guild update

# Guile Package Repository:
# https://github.com/guile-scm

# Структура .scm файла:
(define-module (lib utils)
  #:export (add subtract))

(define (add a b) (+ a b))
(define (subtract a b) (- a b))

# Использование
(use-modules (lib utils))
(add 5 3)  ; 8

5. ПОДВОДНЫЕ КАМНИ УСТАНОВКИ
================================================================================

⚠️ Проблема 1: Несколько версий Guile
   guile-2.2 vs guile-3.0 могут конфликтовать
   
   Решение:
   Использовать guile-3.0 для новых проектов
   Проверять совместимость

⚠️ Проблема 2: C зависимости
   Требуется libguile для C интеграции
   
   Решение:
   sudo apt install guile-3.0-dev

⚠️ Проблема 3: Guild пакетный менеджер
   Меньше пакетов чем npm/pip
   
   Решение:
   Использовать SRFI библиотеки
   Загружать вручную через use-modules

⚠️ Проблема 4: Синтаксис непривычен
   S-выражения, префиксная нотация
   
   Решение:
   Практиковаться ежедневно
   Использовать REPL

⚠️ Проблема 5: Компиляция через guild
   Требуется понимание компилятора
   
   Решение:
   Читать документацию
   Изучать примеры

⚠️ Проблема 6: Разные реализации Scheme
   Guile, Chez, Racket, etc.
   
   Решение:
   Использовать SRFI для совместимости
   Проверять совместимость
