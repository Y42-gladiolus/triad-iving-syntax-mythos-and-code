3. ОПТИМИЗАЦИЯ: КАК УСКОРИТЬ GUILE
================================================================================

Совет 1: Используйте tail-рекурсию
----------------------------------
; МЕДЛЕННО (обычная рекурсия):
(define (sum n)
  (if (<= n 0)
      0
      (+ n (sum (- n 1)))))

; БЫСТРО (tail-рекурсия):
(define (sum-tail n acc)
  (if (<= n 0)
      acc
      (sum-tail (- n 1) (+ acc n))))

; ⚡ Guile оптимизирует tail-рекурсию!

Совет 2: Компилируйте в bytecode
--------------------------------
; МЕДЛЕННО (интерпретация):
guile script.scm

; БЫСТРО (bytecode):
guild compile script.scm
guile script.go

; ⚡ Ускорение: 2-5x для runtime!

Совет 3: Используйте C для hot path
-----------------------------------
; МЕДЛЕННО (Scheme):
(define (sum lst)
  (fold-left + 0 lst))

; БЫСТРО (C):
(define sum (dynamic-func "c_sum" libc))

; ⚡ C функции быстрее!

Совет 4: Избегайте cons в циклах
--------------------------------
; МЕДЛЕННО (cons в цикле):
(define (make-list n)
  (if (= n 0)
      '()
      (cons n (make-list (- n 1)))))

; БЫСТРО (vector):
(define (make-vector n)
  (let ((v (make-vector n)))
    (let loop ((i 0))
      (when (< i n)
        (vector-set! v i i)
        (loop (+ i 1))))
    v))

; ⚡ Vectors быстрее lists!

Совет 5: Используйте SRFI библиотеки
------------------------------------
; МЕДЛЕННО (ручная реализация):
(define (my-map f lst)
  (if (null? lst)
      '()
      (cons (f (car lst)) (my-map f (cdr lst)))))

; БЫСТРО (SRFI):
(use-modules (srfi srfi-1))
(map f lst)

; ⚡ SRFI оптимизированы!

9. БЕНЧМАРК: ПРОВЕРЬТЕ САМИ
================================================================================

Создайте benchmark.scm:
-----------------------
(define (sum-enum n)
  (let loop ((i 0) (acc 0))
    (if (> i n)
        acc
        (loop (+ i 1) (+ acc i)))))

(define start (gettimeofday))
(sum-enum 100000000)
(define end (gettimeofday))

(display "Time: ")
(display (- end start))
(display " ms")
(newline)

Запуск:
-------
guile benchmark.scm
