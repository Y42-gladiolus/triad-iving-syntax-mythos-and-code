3. КОГДА PROLOG НЕ ПОДХОДИТ
================================================================================

✗ Численные вычисления (медленнее Python/C++)
✗ Обработка больших данных (нет эффективных структур)
✗ Графические интерфейсы (слабая поддержка)
✗ Системное программирование
✗ Веб-разработка (хотя возможно)
✗ Оптимизация с ограничениями (MiniZinc лучше)

4. КОГДА PROLOG ИДЕАЛЕН
================================================================================

✓ Экспертные системы
✓ Логический вывод и теоремы
✓ Парсинг и обработка языков
✓ Искусственный интеллект (символический)
✓ Базы знаний
✓ Головоломки и игры (Судоку, шахматы)
✓ Прототипирование логики

6. РЕКОМЕНДАЦИЯ
================================================================================

Prolog учить ЕСЛИ:
✓ Интересует логическое программирование
✓ Нужен символический ИИ
✓ Работаете с экспертными системами
✓ Исследования в области логики/лингвистики

Prolog НЕ учить (или позже) ЕСЛИ:
✓ Только оптимизационные задачи (MiniZinc лучше)
✓ Только численные вычисления (Python лучше)
✓ Нужна работа в индустрии (меньше вакансий)
✓ Веб/мобильная разработка
