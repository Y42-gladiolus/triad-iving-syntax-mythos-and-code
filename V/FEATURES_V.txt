1. АРХИТЕКТУРА И ДИЗАЙН
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    V ARCHITECTURE                                            │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  IMPLEMENTATION:                                                             │
│  • Написан на V (self-hosted)                                               │
│  • Компилируется в C (через C backend)                                      │
│  • Native code generation                                                   │
│  • Кроссплатформенный (Linux, macOS, Windows, FreeBSD)                      │
│                                                                              │
│  DESIGN PHILOSOPHY:                                                          │
│  • Simple (простой синтаксис, легко учить)                                  │
│  • Fast (быстрая компиляция < 1 сек)                                        │
│  • Safe (memory safety, no GC по умолчанию)                                 │
│  • Go-like syntax (но проще и чище)                                         │
│  • No garbage collector (опционально)                                       │
│  • No null (по умолчанию)                                                   │
│  • No exceptions                                                            │
│  • No global variables                                                      │
│                                                                              │
│  KEY FEATURES:                                                               │
│  • Fast compilation (< 1 сек для большинства проектов)                      │
│  • No GC (по умолчанию, опционально можно включить)                         │
│  • Memory safety (без GC)                                                   │
│  • Built-in ORM                                                             │
│  • Built-in web framework                                                   │
│  • Built-in JSON parsing                                                    │
│  • C interoperability (easy FFI)                                            │
│  • Hot code reloading                                                       │
│  • Built-in testing                                                         │
│                                                                              │
│  STATUS (2024):                                                              │
│  • Beta / active development                                                │
│  • Production ready для многих задач                                        │
│  • Community-driven                                                         │
│  • Open source (MIT license)                                                │
│  • Created by Alexander Medvednikov                                         │
│                                                                              │
│  COMPARISON:                                                                 │
│  • vs Go: Быстрее компиляция, нет GC, проще синтаксис                       │
│  • vs Rust: Проще, нет borrow checker, faster compilation                   │
│  • vs C: Безопаснее, современнее, проще                                     │
│  • vs Zig: Похожая философия, V проще для начинающих                        │
│  • vs Crystal: Нет GC, быстрее компиляция                                   │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

2. СИСТЕМА ТИПОВ
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    V TYPE SYSTEM                                             │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  STATIC TYPING:                                                              │
│  • Все типы известны на compile-time                                        │
│  • Type inference (не нужно annotating всё)                                 │
│  • Strong type safety                                                       │
│  • No runtime type errors                                                   │
│                                                                              │
│  BASIC TYPES:                                                                │
│  • i8, i16, i32, i64 (signed integers)                                      │
│  • u8, u16, u32, u64 (unsigned integers)                                    │
│  • f32, f64 (floating point)                                                │
│  • bool (true, false)                                                       │
│  • string (UTF-8 strings)                                                   │
│  • rune (Unicode character)                                                 │
│  • byte (alias for u8)                                                      │
│  • int (platform-dependent)                                                 │
│  • usize (platform-dependent unsigned)                                      │
│                                                                              │
│  COMPOUND TYPES:                                                             │
│  • Arrays: [1, 2, 3]                                                        │
│  • Slices: []int                                                            │
│  • Maps: map[string]int                                                     │
│  • Structs: struct Point { x int }                                          │
│  • Interfaces: interface Reader { read() }                                  │
│  • Enums: enum Color { red green blue }                                     │
│  • Optionals: ?string (может быть error)                                    │
│  • Result: !string (может быть error)                                       │
│                                                                              │
│  NO NULL:                                                                    │
│  • Нет null по умолчанию                                                    │
│  • Optionals для nullable values                                            │
│  • No null pointer exceptions                                               │
│  • Safe by design                                                           │
│                                                                              │
│  TYPE INFERENCE:                                                             │
│  • x := 5  # выводит int                                                    │
│  • x := "hello"  # выводит string                                           │
│  • Works across functions                                                   │
│  • Fast inference                                                           │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

6. CONCURRENCY MODEL
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    V CONCURRENCY                                             │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  GOROUTINES (спавны):                                                        │
│  • spawn fn() для легковесных потоков                                       │
│  • Channels для communication                                               │
│  • Like Go goroutines                                                       │
│  • Thousands of concurrent tasks                                            │
│                                                                              │
│  CHANNELS:                                                                   │
│  • ch := chan int{}                                                         │
│  • ch <- value (send)                                                       │
│  • value := <- ch (receive)                                                 │
│  • Buffered и unbuffered                                                    │
│                                                                              │
│  SELECT:                                                                     │
│  • select {                                                                 │
│  •   case ch1 <- value { }                                                  │
│  •   case value := <- ch2 { }                                               │
│  • }                                                                        │
│                                                                              │
│  NO THREADS:                                                                 │
│  • Нет native threads (по умолчанию)                                        │
│  • Single-threaded с spawn                                                  │
│  • Simpler concurrency model                                                │
│  • No race conditions (если нет shared state)                               │
│                                                                              │
│  ASYNC/AWAIT:                                                                │
│  • async fn() для async функций                                             │
│  • await для等待                                                            │
│  • Non-blocking I/O                                                         │
│  • HTTP server с async                                                      │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
