Linux (Ubuntu/Debian):
----------------------
# Вариант 1: Через официальный скрипт
git clone https://github.com/vlang/v
cd v
make
sudo ln -s /path/to/v/v /usr/local/bin/v

# Вариант 2: Через snap
sudo snap install vlang --classic

# Проверка
v --version

macOS (Homebrew):
-----------------
brew install vlang

# Проверка
v --version

Windows:
--------
# Скачать с https://vlang.io/
# Распаковать архив
# Добавить в PATH

# Проверка в PowerShell:
v --version

Docker (универсально):
----------------------
docker run -it --rm vlang/v v --version

================================================================================
3. БЫСТРЫЙ СТАРТ
================================================================================

# V REPL (нет REPL, компиляция только)
# V компилируется напрямую

# Создание проекта
v init my_project
cd my_project

# Структура проекта:
my_project/
├── main.v         # Главный файл
├── v.mod          # Module file
└── README.md      # Документация

# Простая программа (main.v)
fn main() {
  println('Hello, V!')
}

# Компиляция
v build main.v

# Запуск
./main

# Compile and run
v run main.v

# Hot reload
v run -hot main.v

5. ПОДВОДНЫЕ КАМНИ УСТАНОВКИ
================================================================================

⚠️ Проблема 1: Beta статус
   V в активной разработке (не 1.0 ещё)
   
   Решение:
   Следить за обновлениями
   Использовать latest release

⚠️ Проблема 2: Меньше библиотек
   Экосистема только формируется
   
   Решение:
   Использовать C interop
   Писать свои библиотеки

⚠️ Проблема 3: Документация
   Документация ещё развивается
   
   Решение:
   Читать vlang.io
   Присоединиться к Discord

⚠️ Проблема 4: IDE поддержка
   Ограниченная поддержка
   
   Решение:
   Использовать VS Code + V extension
   Или текстовый редактор + CLI

⚠️ Проблема 5: Windows support
   Windows support ещё улучшается
   
   Решение:
   Использовать Linux/macOS для разработки
   Или использовать WSL

⚠️ Проблема 6: Breaking changes
   Язык быстро развивается (до 1.0)
   
   Решение:
   Использовать specific versions
   Следить за changelog

6. АЛИАСЫ ДЛЯ ТЕРМИНАЛА
================================================================================

# Добавить в ~/.bashrc, ~/.zshrc, ~/.mkshrc

echo "alias v='v'" >> ~/.bashrc
echo "alias v='v'" >> ~/.zshrc

echo "alias vrun='v run'" >> ~/.bashrc
echo "alias vrun='v run'" >> ~/.zshrc

echo "alias vbuild='v build'" >> ~/.bashrc
echo "alias vbuild='v build'" >> ~/.zshrc

echo "alias vfmt='v fmt'" >> ~/.bashrc
echo "alias vfmt='v fmt'" >> ~/.zshrc

# Применить изменения
source ~/.bashrc
source ~/.zshrc
source ~/.mkshrc 2>/dev/null || true
