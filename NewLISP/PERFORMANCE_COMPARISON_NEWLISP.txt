3. ОПТИМИЗАЦИЯ: КАК УСКОРИТЬ NEWLISP
================================================================================

Совет 1: Используйте встроенные функции
---------------------------------------
; МЕДЛЕННО (ручная реализация):
(define (sum list)
  (if (null? list)
    0
    (+ (first list) (sum (rest list)))))

; БЫСТРО (встроенная):
(apply + list)

; ⚡ Встроенные функции оптимизированы!

Совет 2: Используйте let для локальных переменных
-------------------------------------------------
; МЕДЛЕННО (global variables):
(define x 0)
(define y 0)
(setq x 5)
(setq y 10)

; БЫСТРО (local variables):
(let ((x 5) (y 10))
  (+ x y))

; ⚡ Local variables быстрее!

Совет 3: Избегайте лишних cons
------------------------------
; МЕДЛЕННО (создание списков):
(define result '())
(dolist (i (sequence 1 1000))
  (push i result -1))

; БЫСТРО (sequence):
(define result (sequence 1 1000))

; ⚡ Встроенные функции эффективнее!

Совет 4: Используйте context для организации
--------------------------------------------
; МЕДЛЕННО (global namespace):
(define func1 ...)
(define func2 ...)
; Конфликты имён!

; БЫСТРО (context):
(context 'MyModule)
(define (func1 ...) ...)
(define (func2 ...) ...)
(context 'MAIN)

; ⚡ Contexts предотвращают конфликты!

Совет 5: Используйте built-in HTTP/XML/JSON
-------------------------------------------
; МЕДЛЕННО (external libraries):
(import "libcurl.so")
; ... много кода ...

; БЫСТРО (built-in):
(http-get "http://example.com")
(json-parse "{\"key\":\"value\"}")

; ⚡ Built-in функции быстрее!

9. БЕНЧМАРК: ПРОВЕРЬТЕ САМИ
================================================================================

Создайте benchmark.lsp:
-----------------------
#!/usr/bin/newlisp

(define (sum-enum n)
  (apply + (sequence 1 n)))

(define start (time-of-day))
(sum-enum 100000000)
(define end (time-of-day))

(println "Time:" (- end start) "ms")

Запуск:
-------
chmod +x benchmark.lsp
./benchmark.lsp
