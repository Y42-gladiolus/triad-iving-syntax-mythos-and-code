Linux (Ubuntu/Debian):
----------------------

sudo apt update
sudo apt install newlisp newlisp-doc

# Вариант 2: Из исходников

tar xzf v.10.7.5.tar.gz
cd newlisp-10.7.5
./configure
make
sudo make install

# Проверка
newlisp -v

-----------------

# NewLISP REPL
newlisp

# В REPL:
> (+ 2 2)

> (exit)  # Выход

# Создание проекта

; lib/utils.lsp
(define (my-function x)
  (* x 2))

# Использование
#!/usr/bin/newlisp
(load "lib/utils.lsp")
(println (my-function 5))

# NewLISP modules (встроенные)

5. ПОДВОДНЫЕ КАМНИ УСТАНОВКИ
================================================================================

⚠️ Проблема 1: Dynamic scope
   Отличается от lexical scope (Common Lisp/Clojure)
   
   Решение:
   Читать документацию по scope
   Использовать let/letex для локальных переменных

⚠️ Проблема 2: Нет package system
   Нет как ASDF/Quicklisp в Common Lisp
   
   Решение:
   Использовать load для библиотек
   Организовать код в контексты

⚠️ Проблема 3: Mutable по умолчанию
   Не immutable как Clojure
   
   Решение:
   Использовать функциональный стиль
   Избегать мутации когда возможно

⚠️ Проблема 4: Меньше сообщество
   Меньше чем Common Lisp/Clojure
   
   Решение:
   Присоединиться к newlisp forum
   Читать документацию

⚠️ Проблема 5: Windows поддержка
   Может быть сложнее на Windows
   
   Решение:
   Использовать WSL
   Или использовать установщик .exe

⚠️ Проблема 6: Документация
   Меньше чем mainstream языки
   
   Решение:
   Читать официальную документацию
   Изучать примеры на GitHub
