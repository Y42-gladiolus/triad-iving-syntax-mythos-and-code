3. ОПТИМИЗАЦИЯ: КАК УСКОРИТЬ AWK
================================================================================

Совет 1: Используйте BEGIN для инициализации
--------------------------------------------
# МЕДЛЕННО (инициализация в каждой строке):
{if (NR == 1) sum = 0; sum += $1}

# БЫСТРО (инициализация в BEGIN):
BEGIN {sum = 0}
{sum += $1}

# ⚡ BEGIN выполняется один раз!

Совет 2: Используйте next для пропуска
--------------------------------------
# МЕДЛЕННО (вложенные if):
{
    if ($1 !~ /^#/) {
        if ($1 !~ /^[[:space:]]*$/) {
            print $0
        }
    }
}

# БЫСТРО (next):
/^#/ {next}
/^[[:space:]]*$/ {next}
{print $0}

# ⚡ next предотвращает лишнюю обработку!

Совет 3: Избегайте gsub когда можно
-----------------------------------
# МЕДЛЕННО (gsub для каждой строки):
{gsub(/old/, "new"); print}

# БЫСТРО (только когда нужно):
/old/ {gsub(/old/, "new"); print; next}
{print}

# ⚡ gsub имеет оверхед!

Совет 4: Используйте правильные разделители
-------------------------------------------
# МЕДЛЕННО (split в каждой строке):
{split($0, arr, ","); print arr[1]}

# БЫСТРО (FS в BEGIN):
BEGIN {FS = ","}
{print $1}

# ⚡ FS эффективнее split!

Совет 5: Минимизируйте print вызовы
-----------------------------------
# МЕДЛЕННО (много print):
{print $1; print $2; print $3}

# БЫСТРО (один print):
{print $1, $2, $3}

# ⚡ Меньше print = быстрее!

8. ИНТЕГРАЦИЯ AWK С ДРУГИМИ ЯЗЫКАМИ
================================================================================

AWK + bash (pipeline):
----------------------
cat file.txt | awk '{print $1}' | sort | uniq

AWK + Python (через stdin/stdout):
----------------------------------
awk '{print $1}' file.txt | python3 process.py

AWK + grep (комбинация):
------------------------
grep ERROR file.log | awk '{print $1, $2}'

AWK + sed (комбинация):
-----------------------
sed 's/old/new/g' file.txt | awk '{print $1}'

AWK + MiniZinc (через CLI):
---------------------------
awk '{print $1}' data.txt | minizinc model.mzn
