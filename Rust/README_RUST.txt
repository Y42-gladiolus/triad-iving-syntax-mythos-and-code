1. УСТАНОВКА RUST (RUSTUP)
================================================================================

Linux/macOS (официальный способ - рекомендуется):
--------------------------------------------------
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# После установки:
source $HOME/.cargo/env

# Проверка
rustc --version
cargo --version

# Обновление
rustup update

# Установка конкретной версии
rustup install 1.75.0
rustup default 1.75.0

Linux (Ubuntu/Debian - через пакетный менеджер):
------------------------------------------------
sudo apt install rustc cargo
# ⚠️ Версия может быть устаревшей!

macOS (Homebrew):
-----------------
brew install rust

Windows:
--------
1. Скачать rustup-init.exe с https://rustup.rs/
2. Запустить установщик
3. Установить MSVC build tools (предложит автоматически)
4. Проверка в PowerShell: rustc --version

Docker (универсально):
----------------------
docker run -it --rm rust:latest rustc --version

3. БЫСТРЫЙ СТАРТ
================================================================================

# Создание нового проекта
cargo new my_project
cd my_project

# Структура проекта:
my_project/
├── Cargo.toml      # Манифест проекта (зависимости, метаданные)
├── src/
│   └── main.rs     # Точка входа
└── target/         # Скомпилированные файлы (git ignore)

# Запуск проекта
cargo run

# Сборка (релиз)
cargo build --release

# Сборка (debug)
cargo build

# Тесты
cargo test

# Проверка кода (линтер)
cargo clippy

# Форматирование
cargo fmt

# Документация
cargo doc --open

# Запуск без создания проекта (как скрипт)
rustc script.rs
./script

# Или через cargo-script (требует установки)
cargo install cargo-script
cargo script script.rs

4. УПРАВЛЕНИЕ ПАКЕТАМИ (CARGO)
================================================================================

Cargo.toml - манифест проекта:
------------------------------
[package]
name = "my_project"
version = "0.1.0"
edition = "2021"

[dependencies]
serde = "1.0"
serde_json = "1.0"
tokio = { version = "1.0", features = ["full"] }

[dev-dependencies]
criterion = "0.5"  # Только для тестов

[profile.release]
opt-level = 3
lto = true

Основные команды Cargo:
-----------------------
cargo new name          # Создать новый бинарный проект
cargo new --lib name    # Создать библиотеку
cargo build             # Собрать проект
cargo build --release   # Собрать с оптимизациями
cargo run               # Собрать и запустить
cargo test              # Запустить тесты
cargo check             # Быстрая проверка (без компиляции)
cargo clippy            # Линтер (рекомендации)
cargo fmt               # Форматирование кода
cargo doc               # Генерация документации
cargo update            # Обновить зависимости
cargo clean             # Очистить target директорию

# Поиск пакетов
https://crates.io       # Официальный реестр
https://lib.rs          # Альтернативный поиск

5. ПОДВОДНЫЕ КАМНИ УСТАНОВКИ
================================================================================

⚠️ Проблема 1: Долгая первая компиляция
   Первый cargo build может занять 5-30 минут
   
   Решение: Это нормально - компиляция зависимостей
   Последующие сборки используют кэш

⚠️ Проблема 2: Borrow checker ошибки
   Rust не компилирует код с проблемами владения
   
   Решение: Изучить систему владения (ownership)
   Это защита от ошибок, не баг!

⚠️ Проблема 3: Версии зависимостей
   Разные пакеты могут требовать несовместимые версии
   
   Решение: cargo update, cargo tree для анализа

⚠️ Проблема 4: Размер бинарника
   Release бинарники могут быть большими (10-50 MB)
   
   Решение: 
   [profile.release]
   strip = true
   lto = true
   codegen-units = 1

⚠️ Проблема 5: Время компиляции в debug
   Debug сборка медленнее чем интерпретируемые языки
   
   Решение: Использовать cargo check для быстрой проверки
   cargo build --release для финальной сборки

⚠️ Проблема 6: Линукс зависимости
   Некоторые крейты требуют системные библиотеки
   
   Решение: 
   sudo apt install pkg-config libssl-dev

⚠️ Проблема 7: Cross-компиляция
   Компиляция под другую платформу сложна
   
   Решение: Использовать cargo-cross
   cargo install cross
   cross build --target x86_64-unknown-linux-musl

echo "alias cb='cargo build'" >> ~/.zshrc

echo "alias cr='cargo run'" >> ~/.zshrc

echo "alias ct='cargo test'" >> ~/.zshrc

echo "alias cc='cargo check'" >> ~/.zshrc

# Применить изменения
source ~/.bashrc
source ~/.zshrc
source ~/.mkshrc 2>/dev/null || true
