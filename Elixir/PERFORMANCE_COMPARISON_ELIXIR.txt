3. ОПТИМИЗАЦИЯ: КАК УСКОРИТЬ ELIXIR
================================================================================

Совет 1: Используйте Stream для больших данных
----------------------------------------------
# МЕДЛЕННО (Enum):
1..1_000_000
|> Enum.map(&(&1 * 2))
|> Enum.filter(&(&1 > 5))

# БЫСТРО (Stream):
1..1_000_000
|> Stream.map(&(&1 * 2))
|> Stream.filter(&(&1 > 5))
|> Enum.take(10)

# ⚡ Ускорение: 10-100x для больших данных!

Совет 2: Используйте proper patterns
------------------------------------
# МЕДЛЕННО (if/else):
if condition do
  result1
else
  result2
end

# БЫСТРО (pattern matching):
case condition do
  true -> result1
  false -> result2
end

# ⚡ Pattern matching оптимизирован в BEAM!

Совет 3: Используйте :ets для кэша
----------------------------------
# МЕДЛЕННО (Map в GenServer):
GenServer.call(pid, {:get, key})

# БЫСТРО (:ets):
:ets.lookup(:cache, key)

# ⚡ Ускорение: 10-50x для частых запросов!

Совет 4: Используйте NIF для критичного кода
-------------------------------------------
# Elixir код:
def compute(x), do: Native.compute(x)

# Rust/C код (NIF):
# Компилируется в native code

# ⚡ Ускорение: 10-100x для CPU-bound!

Совет 5: Используйте Phoenix LiveView
-------------------------------------
# МЕДЛЕННО (REST API + JS frontend):
API → JSON → JS → Render

# БЫСТРО (LiveView):
Server → HTML → WebSocket → Update

# ⚡ Ускорение разработки: 2-5x!

9. БЕНЧМАРК: ПРОВЕРЬТЕ САМИ
================================================================================

Создайте benchmark.exs:
-----------------------
defmodule Benchmark do
  def sum_enum(n) do
    1..n |> Enum.sum()
  end
  
  def sum_stream(n) do
    1..n |> Stream.map(& &1) |> Enum.sum()
  end
  
  def sum_reduce(n) do
    Enum.reduce(1..n, 0, &(&1 + &2))
  end
end

# Замер времени
{time, result} = :timer.tc(&Benchmark.sum_enum/1, [1_000_000])
IO.puts("Enum: #{time / 1_000_000}秒，结果：#{result}")

Запуск:
-------
elixir benchmark.exs

Сравнение с Python:
-------------------
# Python будет в 2-5 раз медленнее для concurrent задач!
