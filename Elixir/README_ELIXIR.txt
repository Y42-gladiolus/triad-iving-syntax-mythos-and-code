Linux (Ubuntu/Debian):
----------------------
# Шаг 1: Установить зависимости
sudo apt update
sudo apt install git curl wget build-essential libssl-dev libncurses5-dev \
                 unixodbc unixodbc-dev

# Шаг 2: Установить asdf
git clone https://github.com/asdf-vm/asdf.git ~/.asdf --branch v0.12.0
echo '. $HOME/.asdf/asdf.sh' >> ~/.bashrc
echo '. $HOME/.asdf/completions/asdf.bash' >> ~/.bashrc
source ~/.bashrc

# Шаг 3: Добавить плагины
asdf plugin-add erlang
asdf plugin-add elixir

# Шаг 4: Установить версии
asdf install erlang 26.2.0
asdf install elixir 1.16.0

# Шаг 5: Установить глобальные версии
asdf global erlang 26.2.0
asdf global elixir 1.16.0

# Проверка
elixir --version
erl --version

macOS (Homebrew):
-----------------
brew install asdf
# Затем как Linux выше

# Или проще:
brew install elixir
# (установит Erlang автоматически)

Windows:
--------
# Вариант 1: WSL (рекомендуется)
# Установить WSL2 + Ubuntu, затем как Linux выше

# Вариант 2: Native
# Скачать с https://elixir-lang.org/install.html
# Установщик .exe

Docker (универсально):
----------------------
docker run -it --rm elixir:1.16 iex

4. УПРАВЛЕНИЕ ПАКЕТАМИ (HEX + MIX)
================================================================================

# Mix = build tool (как cargo/npm)
# Hex = пакетный менеджер (как crates.io/npm)

# Создание проекта:
mix new my_project
mix new my_project --umbrella  # Для больших проектов

# Основные команды Mix:
mix compile      # Скомпилировать
mix run          # Запустить
mix test         # Тесты
mix deps.get     # Установить зависимости
mix deps.compile # Скомпилировать зависимости
mix format       # Форматирование
mix hex.info     # Информация о пакете

# Hex команды:
mix hex.info package_name     # Информация о пакете
mix hex.search keyword        # Поиск пакетов
mix hex.owner add package     # Добавить владельца
mix hex.docs build            # Построить документацию

# Структура проекта:
my_project/
├── mix.exs          # Конфигурация проекта
├── mix.lock         # Заблокированные версии зависимостей
├── lib/             # Исходный код
│   └── my_project.ex
├── test/            # Тесты
│   └── my_project_test.exs
└── config/          # Конфигурация
    └── config.exs

5. ПОДВОДНЫЕ КАМНИ УСТАНОВКИ
================================================================================

⚠️ Проблема 1: Erlang не установлен
   Elixir требует Erlang/OTP
   
   Решение: 
   asdf plugin-add erlang
   asdf install erlang 26.2.0

⚠️ Проблема 2: Конфликт версий
   Разные проекты требуют разные версии
   
   Решение: asdf manage версии
   asdf local elixir 1.15.0  # Для конкретного проекта

⚠️ Проблема 3: Зависимости не компилируются
   Некоторые пакеты требуют компиляции C
   
   Решение:
   sudo apt install build-essential

⚠️ Проблема 4: ElixirLS не работает в VS Code
   Language server не запускается
   
   Решение:
   mix local.rebar --force
   mix local.hex --force
   Перезапустить VS Code

⚠️ Проблема 5: Unicode в Windows
   Кириллица может не отображаться
   
   Решение:
   Использовать WSL или настроить кодировку UTF-8

⚠️ Проблема 6: Порты заняты (Phoenix)
   Port 4000 уже используется
   
   Решение:
   Изменить порт в config/dev.exs
   или: lsof -ti:4000 | xargs kill
