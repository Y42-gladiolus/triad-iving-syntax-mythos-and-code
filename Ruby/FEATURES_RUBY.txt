1. АРХИТЕКТУРА И ДИЗАЙН
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    RUBY ARCHITECTURE                                         │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  IMPLEMENTATION:                                                             │
│  • Написан на C (MRI - Matz's Ruby Interpreter)                             │
│  • Интерпретируемый (с JIT в Ruby 3+)                                       │
│  • YARV bytecode                                                            │
│  • Кроссплатформенный (Linux, macOS, Windows)                               │
│                                                                              │
│  DESIGN PHILOSOPHY:                                                          │
│  • Developer happiness (главный приоритет)                                  │
│  • Elegant syntax (читаемый и красивый)                                     │
│  • Principle of Least Surprise                                              │
│  • Everything is an object                                                  │
│  • Dynamic typing                                                           │
│  • Duck typing                                                              │
│  • Blocks и iterators                                                       │
│                                                                              │
│  KEY FEATURES:                                                               │
│  • Beautiful syntax (поэтичный код)                                         │
│  • Metaprogramming (мощный и гибкий)                                        │
│  • Blocks и procs (функциональные возможности)                              │
│  • Mixins (modules вместо множественного наследования)                      │
│  • Garbage collected                                                        │
│  • Huge ecosystem (RubyGems)                                                │
│  • Rails framework (web development)                                        │
│                                                                              │
│  STATUS (2024):                                                              │
│  • Stable release (3.x)                                                     │
│  • Production ready                                                         │
│  • Mature ecosystem                                                         │
│  • Open source (Ruby License)                                               │
│  • Created by Yukihiro Matsumoto (Matz)                                     │
│  • 1995+ разработка                                                         │
│                                                                              │
│  COMPARISON:                                                                 │
│  • vs Python: Более элегантный синтаксис, мощнее metaprogramming            │
│  • vs Crystal: Ruby медленнее но зрелее экосистема                          │
│  • vs Java: Проще синтаксис, динамическая типизация                         │
│  • vs Go: Выразительнее, но медленнее                                       │
│  • vs Elixir: Ruby старше, Elixir функциональнее                            │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

2. СИСТЕМА ТИПОВ
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    RUBY TYPE SYSTEM                                          │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  DYNAMIC TYPING:                                                             │
│  • Типы проверяются на runtime                                              │
│  • Нет type annotations (опционально в Ruby 3+)                             │
│  • Flexible и гибкий                                                        │
│  • Duck typing ("если ходит как утка...")                                   │
│                                                                              │
│  BASIC TYPES:                                                                │
│  • Integer (1, -10, 1000000)                                                │
│  • Float (3.14, -0.5)                                                       │
│  • String ("hello", "привет")                                               │
│  • Symbol (:name, :age - immutable strings)                                 │
│  • Boolean (true, false)                                                    │
│  • Nil (nil - единственное ложное значение кроме false)                     │
│  • Array ([1, 2, 3])                                                        │
│  • Hash ({key: value})                                                      │
│  • Range (1..10, 1...10)                                                    │
│  • Regexp (/pattern/)                                                       │
│  • Proc/lambda (блоки кода)                                                 │
│                                                                              │
│  RUBY 3+ TYPE SIGNATURES:                                                    │
│  • RBS (Ruby Signature) для type hints                                      │
│  • Steep для static type checking                                           │
│  • Опционально (backward compatible)                                        │
│  • Better IDE support                                                       │
│                                                                              │
│  DUCK TYPING:                                                                │
│  • Не важно какой тип, важно что может делать                               │
│  • if obj.respond_to?(:method_name)                                         │
│  • Flexible polymorphism                                                    │
│  • No interfaces needed                                                     │
│                                                                              │
│  EVERYTHING IS OBJECT:                                                       │
│  • Числа — объекты                                                          │
│  • Классы — объекты                                                         │
│  • Методы — объекты                                                         │
│  • Nil — объект                                                             │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

3. УПРАВЛЕНИЕ ПАМЯТЬЮ
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    RUBY MEMORY MANAGEMENT                                    │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  GARBAGE COLLECTION:                                                         │
│  • Mark-and-sweep GC                                                        │
│  • Generational GC (Ruby 2.1+)                                              │
│  • Incremental GC (Ruby 2.2+)                                               │
│  • Compacting GC (Ruby 2.3+)                                                │
│                                                                              │
│  GC CHARACTERISTICS:                                                         │
│  • GC паузы (короткие в Ruby 3+)                                            │
│  • Tunable GC parameters                                                    │
│  • GC.stat для мониторинга                                                  │
│  • GC.compact для defragmentation                                           │
│                                                                              │
│  NO MANUAL CONTROL:                                                          │
│  • Нет manual memory management                                             │
│  • Нет malloc/free                                                          │
│  • Автоматическое управление                                                │
│  • No memory leaks (если нет C extensions)                                  │
│                                                                              │
│  MEMORY FOOTPRINT:                                                           │
│  • Moderate to high memory usage                                            │
│  • GC overhead ~20-30%                                                      │
│  • Suitable for most applications                                           │
│  • Not ideal for memory-constrained systems                                 │
│                                                                              │
│  OPTIMIZATIONS:                                                              │
│  • Object pooling                                                           │
│  • String interning                                                         │
│  • Symbol table (immutable strings)                                         │
│  • Copy-on-write for fork                                                   │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

│                                                                              │
│  RUNTIME OPTIONS:                                                            │

│  • ruby --jit script.rb (с JIT)                                             │

│                                                                              │
│  TARGETS:                                                                    │
│  • Linux                                                                    │
│  • macOS                                                                    │
│  • Windows                                                                  │
│  • FreeBSD                                                                  │
│  • Any platform с C compiler                                                │
│                                                                              │
│  STARTUP SPEED:                                                              │
│  • Fast startup (< 100 ms)                                                  │
│  • No compilation delay                                                     │
│  • Perfect for CLI tools                                                    │
│  • Slower than compiled languages                                           │
│                                                                              │

5. ERROR HANDLING
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    RUBY ERROR HANDLING                                       │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  EXCEPTIONS:                                                                 │
│  • raise для создания исключений                                            │
│  • begin/rescue/ensure для обработки                                        │
│  • Exception hierarchy                                                      │
│  • Как Python, Java                                                         │
│                                                                              │
│  EXCEPTION HIERARCHY:                                                        │
│  • Exception (базовый класс)                                                │
│  • StandardError (обычные ошибки)                                           │
│  • RuntimeError (ошибки runtime)                                            │
│  • ArgumentError, TypeError, etc.                                           │
│                                                                              │
│  EXAMPLE:                                                                    │
│  • begin                                                                    │
│  •   risky_operation                                                      │
│  • rescue ArgumentError => e                                                │
│  •   handle_error(e)                                                        │
│  • ensure                                                                   │
│  •   cleanup                                                                │
│  • end                                                                      │
│                                                                              │
│  RETRY:                                                                      │
│  • retry для повторной попытки                                              │
│  • Useful для transient errors                                              │
│  • В rescue блоке                                                           │
│  •retry                                                                       │
│                                                                              │
│  ASSERTIONS:                                                                 │
│  • assert для testing                                                       │
│  • raise unless condition                                                   │
│  • raise if condition                                                       │
│  • Minitest/RSpec для testing                                               │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

6. CONCURRENCY MODEL
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    RUBY CONCURRENCY                                          │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  THREADS:                                                                    │
│  • Native threads (Ruby 1.9+)                                               │
│  • GVL (Global VM Lock) ограничивает parallelism                            │
│  • I/O-bound tasks работают хорошо                                          │
│  • CPU-bound tasks ограничены GVL                                           │
│                                                                              │
│  FIBERS:                                                                     │
│  • Легковесные coroutines                                                   │
│  • Cooperative multitasking                                                 │
│  • Manual yield/resume                                                      │
│  • Like generators в Python                                                 │
│                                                                              │
│  RACTOR (Ruby 3+):                                                           │
│  • Actor model для parallelism                                              │
│  • Обходит GVL ограничение                                                  │
│  • Message passing между ractors                                            │
│  • Thread-safe по дизайну                                                   │
│                                                                              │
│  ASYNC/AWAIT:                                                                │
│  • Async gem для async/await                                                │
│  • Non-blocking I/O                                                         │
│  • Event loop                                                               │
│  • Growing ecosystem                                                        │
│                                                                              │
│  PROCESSES:                                                                  │
│  • fork для создания процессов                                              │
│  • Copy-on-write efficient                                                  │
│  • Process.spawn для контроля                                               │
│  • Inter-process communication                                              │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
