3. ОПТИМИЗАЦИЯ: КАК УСКОРИТЬ RUBY
================================================================================

Совет 1: Используйте Ruby 3+ с YJIT
-----------------------------------
# МЕДЛЕННО (Ruby 2.x):
ruby script.rb

# БЫСТРО (Ruby 3.1+ с YJIT):
ruby --yjit script.rb

# ⚡ YJIT ускоряет в 1.5-3x!

Совет 2: Избегайте лишних аллокаций
-----------------------------------
# МЕДЛЕННО (allocation в loop):
arr = []
1000.times do
  arr << compute_value
end

# БЫСТРО (pre-allocate):
arr = Array.new(1000)
1000.times do |i|
  arr[i] = compute_value
end

# ⚡ Pre-allocation быстрее!

Совет 3: Используйте символы вместо строк для keys
-------------------------------------------------
# МЕДЛЕННО (strings):
hash = {"name" => "John", "age" => 25}

# БЫСТРО (symbols):
hash = {name: "John", age: 25}

# ⚡ Symbols immutable и быстрее!

Совет 4: Используйте Enumerator для lazy evaluation
---------------------------------------------------
# МЕДЛЕННО (eager):
result = [1, 2, 3, 4, 5].map { |x| x * 2 }.select { |x| x > 5 }

# БЫСТРО (lazy):
result = [1, 2, 3, 4, 5].lazy.map { |x| x * 2 }.select { |x| x > 5 }.force

# ⚡ Lazy evaluation экономит память!

Совет 5: Используйте параллелизм для I/O-bound tasks
----------------------------------------------------
# МЕДЛЕННО (последовательно):
urls.each do |url|
  download(url)
end

# БЫСТРО (параллельно):
require 'parallel'
Parallel.each(urls) do |url|
  download(url)
end

# ⚡ Parallel быстрее для I/O!

9. БЕНЧМАРК: ПРОВЕРЬТЕ САМИ
================================================================================

Создайте benchmark.rb:
----------------------
def sum_enum(n)
  sum = 0
  (1..n).each do |i|
    sum += i
  end
  sum
end

start = Time.now
result = sum_enum(100_000_000)
finish = Time.now

puts "Sum: #{result}"
puts "Time: #{(finish - start) * 1000} ms"

Запуск:
-------
ruby benchmark.rb
ruby --yjit benchmark.rb  # С YJIT
