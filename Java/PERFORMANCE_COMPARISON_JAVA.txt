3. ОПТИМИЗАЦИЯ: КАК УСКОРИТЬ JAVA
================================================================================

Совет 1: Используйте Virtual Threads (Java 21+)
-----------------------------------------------
// МЕДЛЕННО (Platform threads):
ExecutorService executor = Executors.newFixedThreadPool(100);

// БЫСТРО (Virtual threads):
ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor();

// ⚡ Ускорение: 10-100x для I/O-bound задач!

Совет 2: Используйте Stream.parallel() для больших данных
---------------------------------------------------------
// МЕДЛЕННО (sequential):
list.stream().map(this::process).toList();

// БЫСТРО (parallel):
list.parallelStream().map(this::process).toList();

// ⚡ Ускорение: 2-8x на multi-core!

Совет 3: Избегайте autoboxing в циклах
--------------------------------------
// МЕДЛЕННО (autoboxing):
Integer sum = 0;
for (int i = 0; i < 1000; i++) {
    sum += i;  // Autoboxing каждый раз!
}

// БЫСТРО (primitive):
int sum = 0;
for (int i = 0; i < 1000; i++) {
    sum += i;
}

// ⚡ Ускорение: 5-10x для интенсивных циклов!

Совет 4: Используйте StringBuilder для конкатенации
---------------------------------------------------
// МЕДЛЕННО (String конкатенация):
String result = "";
for (int i = 0; i < 1000; i++) {
    result += i;  // Создаёт новый String каждый раз!
}

// БЫСТРО (StringBuilder):
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 1000; i++) {
    sb.append(i);
}
String result = sb.toString();

// ⚡ Ускорение: 100-1000x для больших строк!

Совет 5: Используйте JIT warm-up
--------------------------------
// Запустите несколько раз перед бенчмарком:
for (int i = 0; i < 1000; i++) {
    benchmark();  // Warm-up
}

// Затем измеряйте:
long start = System.nanoTime();
benchmark();
long end = System.nanoTime();

// ⚡ JIT оптимизирует после нескольких запусков!

9. БЕНЧМАРК: ПРОВЕРЬТЕ САМИ
================================================================================

Создайте Benchmark.java:
------------------------
public class Benchmark {
    public static long sumEnum(long n) {
        long sum = 0;
        for (long i = 1; i <= n; i++) {
            sum += i;
        }
        return sum;
    }
    
    public static void main(String[] args) {
        // Warm-up
        for (int i = 0; i < 100; i++) {
            sumEnum(1_000_000);
        }
        
        // Benchmark
        long start = System.nanoTime();
        long result = sumEnum(1_000_000_000L);
        long end = System.nanoTime();
        
        System.out.println("Sum: " + result);
        System.out.println("Time: " + (end - start) / 1_000_000 + " ms");
    }
}

Компиляция и запуск:
--------------------
javac Benchmark.java
java Benchmark

# С оптимизациями:
java -XX:+UseG1GC -Xmx2g Benchmark

Сравнение с другими:
--------------------
# Java будет быстрее Python, медленнее C/Rust
# Но после warm-up разница меньше!
