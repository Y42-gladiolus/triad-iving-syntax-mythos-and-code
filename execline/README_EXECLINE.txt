Linux (Ubuntu/Debian):
----------------------
# Вариант 1: Через apt (может быть старая версия)
sudo apt update
sudo apt install execline

# Вариант 2: Из исходников (рекомендуется!)
wget https://skarnet.org/software/execline/execline-2.9.3.0.tar.gz
tar xzf execline-2.9.3.0.tar.gz
cd execline-2.9.3.0
./configure --prefix=/usr
make
sudo make install

# Требуется skalibs (библиотека)
wget https://skarnet.org/software/skalibs/skalibs-2.14.1.1.tar.gz
tar xzf skalibs-2.14.1.1.tar.gz
cd skalibs-2.14.1.1
./configure --prefix=/usr
make
sudo make install

# Проверка
execlineb -v

3. БЫСТРЫЙ СТАРТ
================================================================================

# execline не имеет REPL (не интерактивный!)
# Только скрипты

# Запуск скрипта
execlineb script

# Или через shebang
#!/command/execlineb
importas -u PATH PATH
posix-chronic
echo "Hello, execline!"

# Создание проекта
mkdir my_project && cd my_project

# Структура проекта:
my_project/
├── run              # Главный скрипт
├── finish           # Finish скрипт
├── log/             # Логирование
│   └── run
└── env/             # Переменные окружения

# Простой скрипт
#!/command/execlineb -S
echo "Hello, World!"

# Запуск
chmod +x run
./run

5. ПОДВОДНЫЕ КАМНИ УСТАНОВКИ
================================================================================

⚠️ Проблема 1: Требуется skalibs
   execline требует библиотеку skalibs
   
   Решение:
   Установить skalibs перед execline

⚠️ Проблема 2: Нет REPL
   Нельзя тестировать интерактивно
   
   Решение:
   Писать скрипты и запускать
   Использовать echo для отладки

⚠️ Проблема 3: Синтаксис непривычен
   Нет переменных как в bash
   
   Решение:
   Читать документацию skarnet.org
   Изучать примеры скриптов

⚠️ Проблема 4: Мало документации
   Меньше чем у bash
   
   Решение:
   Читать официальную документацию
   Изучать примеры на GitHub

⚠️ Проблема 5: Только Unix
   Не работает на Windows
   
   Решение:
   Использовать WSL или Docker

⚠️ Проблема 6: Интеграция с systemd
   Требуется настройка сервисов
   
   Решение:
   Читать s6/systemd integration guide
   Использовать s6-rc для управления
