1. УСТАНОВКА R И RSTUDIO
================================================================================

Linux (Ubuntu/Debian):
----------------------
# Базовая установка R
sudo apt update
sudo apt install r-base r-base-dev

# Проверка версии
R --version

# Установка RStudio (рекомендуется!)
wget https://download1.rstudio.org/electron/jammy/amd64/rstudio-2024.04.0-735-amd64.deb
sudo apt install ./rstudio-2024.04.0-735-amd64.deb

# Альтернатива: через пакетный менеджер
sudo apt install rstudio

macOS (Homebrew):
-----------------
brew install r
brew install --cask rstudio

# Проверка
R --version

Windows:
--------
1. Скачать R с https://cran.r-project.org/bin/windows/base/
2. Установить R
3. Скачать RStudio с https://posit.co/download/rstudio-desktop/
4. Установить RStudio
5. Проверка в PowerShell: R --version

Docker (универсально):
----------------------
docker run -ti --rm rocker/rstudio

4. УПРАВЛЕНИЕ ПАКЕТАМИ
================================================================================

# Установка пакета
install.packages("ggplot2")

# Загрузка пакета в сессию
library(ggplot2)

# Обновление всех пакетов
update.packages()

# Просмотр установленных пакетов
installed.packages()

# Удаление пакета
remove.packages("package_name")

# Установка нескольких пакетов
install.packages(c("ggplot2", "dplyr", "tidyr"))

# Установка с CRAN (указать репозиторий)
install.packages("package", repos="https://cloud.r-project.org")

# Установка из GitHub
install.packages("devtools")
devtools::install_github("user/repo")

# Установка из Bioconductor (биоинформатика)
if (!require("BiocManager", quietly = TRUE))
    install.packages("BiocManager")
BiocManager::install("package")

5. ПОДВОДНЫЕ КАМНИ УСТАНОВКИ
================================================================================

⚠️ Проблема 1: Нет прав на установку пакетов
   Ошибка: "cannot open file ... Permission denied"
   
   Решение 1: Установить в домашнюю директорию
   install.packages("package", lib="~/R/library")
   
   Решение 2: Исправить права (Linux)
   sudo chown -R $USER:$USER /usr/local/lib/R/site-library

⚠️ Проблема 2: Зависимости не устанавливаются
   Ошибка: "dependency is not available"
   
   Решение: Установить зависимости явно
   install.packages("package", dependencies=TRUE)

⚠️ Проблема 3: Компиляция пакетов fails (Linux)
   Ошибка: "g++: not found" или "make: not found"
   
   Решение: Установить build tools
   sudo apt install build-essential
   sudo apt install r-base-dev

⚠️ Проблема 4: Медленная загрузка CRAN
   Решение: Выбрать быстрый mirror
   install.packages("package", repos="https://cloud.r-project.org")

⚠️ Проблема 5: Конфликт версий пакетов
   Решение: Использовать packrat или renv
   install.packages("renv")
   renv::init()

⚠️ Проблема 6: Кодировка (Windows, кириллица)
   Решение: Сохранять файлы в UTF-8
   В RStudio: Tools → Global Options → Code → Saving → UTF-8

6. АЛИАСЫ ДЛЯ ТЕРМИНАЛА
================================================================================

# Добавить в ~/.bashrc, ~/.zshrc, ~/.mkshrc

echo "alias r='R --vanilla'" >> ~/.bashrc
echo "alias r='R --vanilla'" >> ~/.zshrc

# Для запуска скриптов
echo "alias rs='Rscript'" >> ~/.bashrc
echo "alias rs='Rscript'" >> ~/.zshrc

# Применить изменения
source ~/.bashrc
source ~/.zshrc
source ~/.mkshrc 2>/dev/null || true

# Теперь:
# r   → запуск R интерпретатора
# rs script.R → запуск скрипта
