1. АРХИТЕКТУРА И ДИЗАЙН
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    CRYSTAL ARCHITECTURE                                      │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  IMPLEMENTATION:                                                             │
│  • Написан на Crystal (self-hosted)                                         │
│  • Компилируется в LLVM IR                                                  │
│  • Native code generation                                                   │
│  • Кроссплатформенный (Linux, macOS, Windows)                               │
│                                                                              │
│  DESIGN PHILOSOPHY:                                                          │
│  • Ruby-like syntax (приятный и читаемый)                                   │
│  • C-level performance (компилируется в native code)                        │
│  • Static typing с type inference                                           │
│  • Garbage collected (но эффективный GC)                                    │
│  • No runtime overhead                                                      │
│  • Productivity + Performance                                               │
│                                                                              │
│  KEY FEATURES:                                                               │
│  • Ruby-like syntax                                                         │
│  • Type inference (не нужно annotating всё)                                 │
│  • Macros (metaprogramming)                                                 │
│  • Fibers (легковесные потоки)                                              │
│  • Channels (communication между fibers)                                    │
│  • Built-in HTTP server                                                     │
│  • JSON/YAML из коробки                                                     │
│  • C bindings (easy FFI)                                                    │
│                                                                              │
│  STATUS (2024):                                                              │
│  • Stable release (1.x)                                                     │
│  • Production ready                                                         │
│  • Community-driven                                                         │
│  • Open source (Apache 2.0 license)                                         │
│  • Created by Ary Borenszweig                                               │
│                                                                              │
│  COMPARISON:                                                                 │
│  • vs Ruby: 1000x быстрее, static typing, compiled                          │
│  • vs Python: 100x быстрее, static typing, compiled                         │
│  • vs Go: Similar performance, nicer syntax, GC                             │
│  • vs Rust: Simpler, GC, less safety guarantees                             │
│  • vs Nim: Similar goals, Crystal more mature                               │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

2. СИСТЕМА ТИПОВ
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    CRYSTAL TYPE SYSTEM                                       │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  STATIC TYPING:                                                              │
│  • Все типы известны на compile-time                                        │
│  • Type inference (не нужно annotating)                                     │
│  • Strong type safety                                                       │
│  • No runtime type errors                                                   │
│                                                                              │
│  BASIC TYPES:                                                                │
│  • Int8, Int16, Int32, Int64 (signed integers)                              │
│  • UInt8, UInt16, UInt32, UInt64 (unsigned integers)                        │
│  • Float32, Float64 (floating point)                                        │
│  • Bool (true, false)                                                       │
│  • Char (Unicode character)                                                 │
│  • String (UTF-8 strings)                                                   │
│  • Symbol (:symbol, immutable strings)                                      │
│  • Nil (nil value)                                                          │
│  • Pointer (C pointers)                                                     │
│                                                                              │
│  COMPOUND TYPES:                                                             │
│  • Arrays: [1, 2, 3]                                                        │
│  • Hashes: {"key" => "value"}                                               │
│  • Tuples: {1, "a", true}                                                   │
│  • Named tuples: {name: "John", age: 25}                                    │
│  • Structs: struct Point; end                                               │
│  • Classes: class Person; end                                               │
│  • Modules: module Comparable; end                                          │
│  • Enums: enum Color; end                                                   │
│  • Unions: Int32 | String                                                   │
│  • Nilable: String? (String | Nil)                                          │
│                                                                              │
│  TYPE INFERENCE:                                                             │
│  • x = 5  # выводит Int32                                                   │
│  • x = "hello"  # выводит String                                            │
│  • Works across functions                                                   │
│  • Fast inference                                                           │
│                                                                              │
│  NILABLE TYPES:                                                              │
│  • String? = String | Nil                                                   │
│  • Nil checks обязательны                                                   │
│  • No null pointer exceptions                                               │
│  • Safe by design                                                           │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

3. УПРАВЛЕНИЕ ПАМЯТЬЮ
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    CRYSTAL MEMORY MANAGEMENT                                 │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  GARBAGE COLLECTION:                                                         │
│  • Boehm-Demers-Weiser GC                                                   │
│  • Conservative GC                                                          │
│  • No manual memory management                                              │
│  • No memory leaks (если нет C pointers)                                    │
│                                                                              │
│  GC CHARACTERISTICS:                                                         │
│  • Stop-the-world паузы (короткие)                                          │
│  • Tunable GC parameters                                                    │
│  • Low memory overhead                                                      │
│  • Predictable performance                                                  │
│                                                                              │
│  MANUAL CONTROL:                                                             │
│  • GC.disable для critical sections                                         │
│  • GC.collect для manual collection                                         │
│  • Pointer для manual memory                                                │
│  • C interop с memory management                                            │
│                                                                              │
│  NO REFERENCE COUNTING:                                                      │
│  • Нет reference counting (как Python)                                      │
│  • Нет circular reference problems                                          │
│  • Simpler than reference counting                                          │
│  • Better performance                                                       │
│                                                                              │
│  MEMORY FOOTPRINT:                                                           │
│  • Moderate memory usage                                                    │
│  • GC overhead ~10-20%                                                      │
│  • Suitable for most applications                                           │
│  • Predictable memory usage                                                 │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

│  • end                                                                      │
│                                                                              │
│  RESCUE:                                                                     │
│  • begin                                                                    │
│  •   risky_operation                                                      │
│  • rescue ex : DivisionByZero                                               │
│  •   handle_error(ex)                                                       │
│  • end                                                                      │
│                                                                              │
│  ASSERTIONS:                                                                 │
│  • assert(condition)                                                        │
│  • Compile-time assertions                                                  │
│  • Runtime assertions                                                       │
│  • Spec testing framework                                                   │
│                                                                              │

6. CONCURRENCY MODEL
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    CRYSTAL CONCURRENCY                                       │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  FIBERS (COROUTINES):                                                        │
│  • Легковесные потоки                                                       │
│  • Cooperative multitasking                                                 │
│  • Thousands of fibers                                                      │
│  • Like Go goroutines но cooperative                                        │
│                                                                              │
│  CHANNELS:                                                                   │
│  • Communication между fibers                                               │
│  • Type-safe channels                                                       │
│  • Buffered и unbuffered                                                    │
│  • Select для multiple channels                                             │
│                                                                              │
│  SPAWN:                                                                      │
│  • spawn { do_something } для fiber                                         │
│  • Fiber.yield для pause                                                    │
│  • Fiber.join для等待                                                       │
│  • Easy concurrency                                                         │
│                                                                              │
│  NO THREADS:                                                                 │
│  • Нет native threads (по умолчанию)                                        │
│  • Single-threaded с fibers                                                 │
│  • Simpler concurrency model                                                │
│  • No race conditions (если нет shared state)                               │
│                                                                              │
│  ASYNC PROGRAMMING:                                                          │
│  • Async/await через fibers                                                 │
│  • Non-blocking I/O                                                         │
│  • HTTP server с async                                                      │
│  • Database connections с async                                             │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
