3. ОПТИМИЗАЦИЯ: КАК УСКОРИТЬ JANET
================================================================================

Совет 1: Используйте array вместо table для числовых индексов
------------------------------------------------------------
# МЕДЛЕННО (table):
(def t @{0 1 1 2 2 3})

# БЫСТРО (array):
(def arr @[1 2 3])

# ⚡ Array быстрее для последовательного доступа!

Совет 2: Избегайте создания объектов в циклах
---------------------------------------------
# МЕДЛЕННО (создание в цикле):
(each [i (range 1000)]
  (def obj {:i i}))

# БЫСТРО (переиспользование):
(def obj {})
(each [i (range 1000)]
  (table/put obj :i i))

# ⚡ Меньше аллокаций = быстрее!

Совет 3: Используйте tail-рекурсию
----------------------------------
# МЕДЛЕННО (обычная рекурсия):
(defn sum [n]
  (if (<= n 0)
    0
    (+ n (sum (- n 1)))))

# БЫСТРО (tail-рекурсия):
(defn sum [n &opt acc]
  (if (<= n 0)
    (or acc 0)
    (sum (- n 1) (+ (or acc 0) n))))

# ⚡ Janet оптимизирует tail-рекурсию!

Совет 4: Используйте fibers для I/O
-----------------------------------
# МЕДЛЕННО (блокирующий I/O):
(def data (http/get url))

# БЫСТРО (fiber):
(def f (fiber/new (http/get url)))
(fiber/resume f)

# ⚡ Не блокирует выполнение!

Совет 5: Compile в байт-код
---------------------------
# МЕДЛЕННО (интерпретация):
janet script.janet

# БЫСТРО (байт-код):
janet -c script.janet
janet script.jimage

# ⚡ Ускорение: 2-5x!

9. БЕНЧМАРК: ПРОВЕРЬТЕ САМИ
================================================================================

Создайте benchmark.janet:
-------------------------
(defn sum-enum [n]
  (var sum 0)
  (each [i (range 1 (+ n 1))]
    (set sum (+ sum i)))
  sum)

(def start (os/time))
(def result (sum-enum 100000000))
(def finish (os/time))

(print "Sum:" result)
(print "Time:" (- finish start) "seconds")

Запуск:
-------
janet benchmark.janet

Сравнение с компиляцией:
------------------------
janet -c benchmark.janet
janet benchmark.jimage

# Байт-код будет быстрее!
