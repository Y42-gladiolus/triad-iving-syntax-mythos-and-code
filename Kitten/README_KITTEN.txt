┌──────────────────────────────────────────────────────────────────────────────┐
│                    KITTEN = СОВРЕМЕННЫЙ FORTH + ТИПЫ                         │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  Kitten используется:                                                        │
│  • Системное программирование                                               │
│  • Встраиваемые системы                                                     │
│  • Компиляторы и инструменты                                                │
│  • Образование (понимание стека + типов)                                    │

Linux (Ubuntu/Debian):
----------------------
# Вариант 1: Из исходников (рекомендуется!)
git clone https://github.com/evincarofautumn/kitten.git
cd kitten
make
sudo make install

# Вариант 2: Через Cargo (если есть Rust)
cargo install kitten-lang

# Проверка
kitten --version

3. БЫСТРЫЙ СТАРТ
================================================================================

# Kitten REPL
kitten

# В REPL:
Kitten> 2 2 +
→ 4 : Int
Kitten> exit  # Выход

# Запуск скрипта
kitten script.ktn

# Компиляция в C
kitten --compile script.ktn
gcc -o script script.c
./script

# Создание проекта
mkdir my_project && cd my_project

# Структура проекта:
my_project/
├── main.ktn       # Главный файл
├── lib/           # Библиотеки
│   └── utils.ktn
├── tests/         # Тесты
└── README.md      # Документация

5. ПОДВОДНЫЕ КАМНИ УСТАНОВКИ
================================================================================

⚠️ Проблема 1: Зависимости для компиляции
   Требуется GCC и make
   
   Решение:
   sudo apt install build-essential

⚠️ Проблема 2: Rust зависимости
   Некоторые версии требуют Rust
   
   Решение:
   curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

⚠️ Проблема 3: Stack-based нотация непривычна
   2 2 + вместо 2 + 2
   
   Решение:
   Практиковаться ежедневно
   Думать в терминах стека

⚠️ Проблема 4: Мало документации
   Проект менее популярен чем Forth
   
   Решение:
   Читать официальную документацию
   Изучать примеры на GitHub

⚠️ Проблема 5: Типизация строгая
   Ошибки типов на compile-time
   
   Решение:
   Внимательно следить за типами
   Использовать type inference

⚠️ Проблема 6: Молодой язык
   Меньше стабильности чем Forth
   
   Решение:
   Использовать стабильные релизы
   Следить за обновлениями

echo "alias ktc='kitten --compile'" >> ~/.zshrc

echo "alias ktr='kitten --run'" >> ~/.bashrc
echo "alias ktr='kitten --run'" >> ~/.zshrc

# Применить изменения
source ~/.bashrc
source ~/.zshrc
source ~/.mkshrc 2>/dev/null || true
