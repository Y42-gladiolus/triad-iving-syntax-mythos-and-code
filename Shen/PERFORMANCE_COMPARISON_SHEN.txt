3. ОПТИМИЗАЦИЯ: КАК УСКОРИТЬ SHEN
================================================================================

Совет 1: Используйте type signatures
------------------------------------
; МЕДЛЕННО (без типов):
(define add
  A B -> (+ A B))

; БЫСТРО (с типами):
(define add : number --> number --> number
  A B -> (+ A B))

; ⚡ Компилятор лучше оптимизирует!

Совет 2: Используйте tail-рекурсию
----------------------------------
; МЕДЛЕННО (обычная рекурсия):
(define factorial
  N -> (if (<= N 1)
           1
           (* N (factorial (- N 1)))))

; БЫСТРО (tail-рекурсия):
(define factorial-tail
  N Acc -> (if (<= N 1)
             Acc
             (factorial-tail (- N 1) (* Acc N))))

; ⚡ Shen оптимизирует tail-рекурсию!

Совет 3: Используйте pattern matching
-------------------------------------
; МЕДЛЕННО (if/else):
(define describe
  X -> (if (= X 0)
           "zero"
           (if (= X 1)
               "one"
               "other")))

; БЫСТРО (pattern matching):
(define describe
  0 -> "zero"
  1 -> "one"
  _ -> "other")

; ⚡ Pattern matching оптимизирован!

Совет 4: Компилируйте в release mode
------------------------------------
; МЕДЛЕННО (debug):
shen script.shen

; БЫСТРО (release):
shen --optimize script.shen

; ⚡ Ускорение: 5-10x для runtime!

Совет 5: Используйте встроенные функции
---------------------------------------
; МЕДЛЕННО (ручная реализация):
(define my-length
  List -> (if (null List)
              0
              (+ 1 (my-length (tail List)))))

; БЫСТРО (встроенная):
(length List)

; ⚡ Встроенные функции оптимизированы!

9. БЕНЧМАРК: ПРОВЕРЬТЕ САМИ
================================================================================

Создайте benchmark.shen:
------------------------
(define sum-enum
  : number --> number
  N -> (let Acc 0
         (for ((I 1 N))
            (set! Acc (+ Acc I)))
         Acc))

(setq Start (time))
(sum-enum 100000000)
(setq End (time))

(print "Time: " (- End Start) " ms")
(quit)

Запуск:
-------
shen benchmark.shen
