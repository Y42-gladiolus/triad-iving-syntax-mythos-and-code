3. ОПТИМИЗАЦИЯ: КАК УСКОРИТЬ FACTOR
================================================================================

Совет 1: Используйте inline слова
---------------------------------
\ МЕДЛЕННО (вызов слова):
: add ( a b -- c )
    + ;

\ БЫСТРО (inline):
INLINE: add ( a b -- c )
    + ;

\ ⚡ Устраняет overhead вызова!

Совет 2: Используйте встроенные комбинаторы
-------------------------------------------
\ МЕДЛЕННО (ручная реализация):
: sum ( seq -- n )
    0 swap [ + ] reduce ;

\ БЫСТРО (встроенный):
sum    ! Встроен в stdlib

\ ⚡ Встроенные комбинаторы оптимизированы!

Совет 3: Избегайте аллокаций в циклах
-------------------------------------
\ МЕДЛЕННО (создание в цикле):
: bad ( n -- )
    n [ V{ } clone swap push ] times ;

\ БЫСТРО (переиспользование):
: good ( n -- )
    V{ } clone :> v
    n [ v swap push ] times ;

\ ⚡ Меньше аллокаций = быстрее!

Совет 4: Используйте compiler hints
-----------------------------------
\ МЕДЛЕННО (без подсказок):
: process ( n -- )
    n 2 * ;

\ БЫСТРО (с подсказками):
: process ( n -- n )
    n 2 * ; inline foldable ;

\ ⚡ Compiler может лучше оптимизировать!

Совет 5: Компилируйте в образ
-----------------------------
\ МЕДЛЕННО (интерпретация):
factor script.factor

\ БЫСТРО (компиляция в образ):
factor -image=script.image script.factor
./factor -run=script

\ ⚡ Ускорение: 2-5x для startup!
