Linux (Ubuntu/Debian):
----------------------
# Вариант 1: Через cabal (Haskell)
sudo apt install cabal-install
cabal update
cabal install dhall

# Вариант 2: Бинарный релиз (рекомендуется!)
wget https://github.com/dhall-lang/dhall-haskell/releases/download/dhall-1.41.2/dhall-1.41.2-x86_64-linux.tar.bz2
tar xjf dhall-1.41.2-x86_64-linux.tar.bz2
sudo cp dhall-1.41.2-x86_64-linux/bin/dhall /usr/local/bin/

# Вариант 3: Через Nix
nix-env -iA nixpkgs.dhall

# Проверка
dhall --version

macOS (Homebrew):
-----------------
brew install dhall

# Проверка
dhall --version

Windows:
--------
# Скачать бинарный релиз с GitHub
# https://github.com/dhall-lang/dhall-haskell/releases
# Добавить в PATH

# Проверка в PowerShell:
dhall --version

Docker (универсально):
----------------------
docker run -it --rm dhall/dhall dhall --version

3. БЫСТРЫЙ СТАРТ
================================================================================

# Dhall REPL
dhall

# В REPL:
⊢ 2 + 2
4

⊢ :quit  # Выход

# Компиляция в JSON
dhall-to-json < input.dhall > output.json

# Компиляция в YAML
dhall-to-yaml < input.dhall > output.yaml

# Типизация
dhall type < input.dhall

# Нормализация
dhall normalize < input.dhall

# Создание проекта
mkdir my_project && cd my_project

# Структура проекта:
my_project/
├── config.dhall     # Конфигурация
├── types.dhall      # Типы
├── packages.dhall   # Зависимости
└── output.json      # Скомпилированный JSON

let config = ./config.dhall

# Кэширование импортов
dhall cache

# Clear cache
dhall cache clear

# Структура пакета:
# dhall-package.dhall

, version = "1.0.0"

5. ПОДВОДНЫЕ КАМНИ УСТАНОВКИ
================================================================================

⚠️ Проблема 1: Haskell зависимости
   Dhall написан на Haskell
   
   Решение:
   Использовать бинарный релиз
   Не компилировать из исходников

⚠️ Проблема 2: Network импорты
   Импорт по URL может быть медленным
   
   Решение:
   dhall cache  # Кэшировать импорты
   Использовать локальные файлы

⚠️ Проблема 3: Нет Turing-полноты
   Нельзя писать произвольный код
   
   Решение:
   Это фича, не баг!
   Для конфигурации это правильно

⚠️ Проблема 4: Мало документации
   Молодой проект (2017+)
   
   Решение:
   Читать официальную документацию
   Изучать примеры на GitHub

⚠️ Проблема 5: Типы могут быть сложны
   Для YAML пользователей ново
   
   Решение:
   Начать с простых типов
   Постепенно усложнять

⚠️ Проблема 6: Интеграция с CI/CD
   Требуется настройка пайплайнов
   
   Решение:
   Использовать dhall-to-json в CI
   Автоматизировать компиляцию
