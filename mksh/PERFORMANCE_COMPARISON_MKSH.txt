3. ОПТИМИЗАЦИЯ: КАК УСКОРИТЬ MKSH
================================================================================

Совет 1: Используйте встроенные команды
---------------------------------------
# МЕДЛЕННО (внешние команды):
cat file.txt | grep pattern | wc -l

# БЫСТРО (встроенные):
grep -c pattern file.txt

# ⚡ Меньше процессов = быстрее!

Совет 2: Избегайте subshells
----------------------------
# МЕДЛЕННО (subshell):
COUNT=$(cat file.txt | wc -l)

# БЫСТРО (встроенные):
COUNT=$(wc -l < file.txt)

# ⚡ Subshells имеют оверхед!

Совет 3: Используйте case вместо if
-----------------------------------
# МЕДЛЕННО (multiple if):
if [ "$VAR" = "start" ]; then
    echo "start"
elif [ "$VAR" = "stop" ]; then
    echo "stop"
elif [ "$VAR" = "restart" ]; then
    echo "restart"
fi

# БЫСТРО (case):
case "$VAR" in
    start) echo "start" ;;
    stop) echo "stop" ;;
    restart) echo "restart" ;;
esac

# ⚡ case эффективнее для множественных проверок!

Совет 4: Минимизируйте external commands
----------------------------------------
# МЕДЛЕННО (external):
DIRNAME=$(dirname "$FILE")
BASENAME=$(basename "$FILE")

# БЫСТРО (parameter expansion):
DIRNAME="${FILE%/*}"
BASENAME="${FILE##*/}"

# ⚡ Parameter expansion быстрее!

Совет 5: Используйте set -e для надёжности
------------------------------------------
# МЕДЛЕННО (проверка каждой команды):
command1
if [ $? -ne 0 ]; then
    exit 1
fi
command2

# БЫСТРО (set -e):
set -e
command1
command2

# ⚡ set -e автоматически выходит при ошибке!

---------------------------------

----------------------
#!/bin/mksh
cat file.txt | awk '{print $1}'

mksh + Perl (комбинация):
-------------------------
#!/bin/mksh
perl -lane 'print $F[0]' file.txt

----------------------------

----------------------
#!/bin/mksh
set -e

COUNT=0
for i in $(seq 1 10000); do
    COUNT=$((COUNT + 1))
done

Запуск:
-------
chmod +x benchmark.sh
time ./benchmark.sh
