1. АРХИТЕКТУРА И ДИЗАЙН
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    GO ARCHITECTURE                                           │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  IMPLEMENTATION:                                                             │
│  • Написан на Go и C (self-hosted)                                          │
│  • Компилируется в native code                                              │
│  • Статическая линковка (single binary)                                     │
│  • Кроссплатформенный (Linux, macOS, Windows, BSD)                          │
│                                                                              │
│  DESIGN PHILOSOPHY:                                                          │
│  • Simple (простой синтаксис, мало keywords)                                │
│  • Fast compilation (< 1 сек для большинства проектов)                      │
│  • Explicit (явный код, мало магии)                                         │
│  • Concurrent (goroutines и channels из коробки)                            │
│  • Pragmatic (решает реальные задачи, не академический)                     │
│                                                                              │
│  KEY FEATURES:                                                               │
│  • Goroutines (легковесные потоки, ~2KB stack)                              │
│  • Channels (CSP-style communication)                                       │
│  • Garbage collected (но эффективный GC)                                    │
│  • Interfaces (implicit, duck typing)                                       │
│  • Defer (автоматическая очистка ресурсов)                                  │
│  • Built-in testing framework                                               │
│  • Rich standard library                                                    │
│                                                                              │
│  STATUS (2024):                                                              │
│  • Stable release (1.22+)                                                   │
│  • Production ready                                                         │
│  • Huge ecosystem (cloud, DevOps, backend)                                  │
│  • Open source (BSD license)                                                │
│  • Created by Google (Robert Griesemer, Rob Pike, Ken Thompson)             │
│  • 2009+ разработка                                                         │
│                                                                              │
│  COMPARISON:                                                                 │
│  • vs Python: Go быстрее, статическая типизация, concurrent                 │
│  • vs Java: Go проще, быстрее компиляция, меньше boilerplate                │
│  • vs C++: Go безопаснее, проще, есть GC                                    │
│  • vs Rust: Go проще, быстрее компиляция, но меньше safety                  │
│  • vs Node.js: Go быстрее, true parallelism, type-safe                      │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

3. УПРАВЛЕНИЕ ПАМЯТЬЮ
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    GO MEMORY MANAGEMENT                                      │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  GARBAGE COLLECTION:                                                         │
│  • Concurrent GC (не останавливает программу)                               │
│  • Low latency (< 100 μs pause times)                                       │
│  • Tri-color mark-and-sweep                                                 │
│  • Automatic memory management                                              │
│                                                                              │
│  GC CHARACTERISTICS:                                                         │
│  • GC паузы очень короткие (< 1 ms)                                         │
│  • Tunable через GOGC environment variable                                  │
│  • Предсказуемая производительность                                         │
│  • Подходит для real-time систем                                            │
│                                                                              │
│  NO MANUAL CONTROL:                                                          │
│  • Нет malloc/free (кроме unsafe)                                           │
│  • Автоматическое управление                                                │
│  • No memory leaks (если нет goroutine leaks)                               │
│  • Escape analysis (stack vs heap)                                          │
│                                                                              │
│  MEMORY FOOTPRINT:                                                           │
│  • Moderate memory usage                                                    │
│  • GC overhead ~10-20%                                                      │
│  • Suitable for most applications                                           │
│  • Efficient for concurrent workloads                                       │
│                                                                              │
│  ESCAPE ANALYSIS:                                                            │
│  • Компилятор решает: stack или heap                                        │
│  • Stack allocation быстрее                                                 │
│  • Можно проверить: go build -gcflags="-m"                                  │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

5. ERROR HANDLING
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    GO ERROR HANDLING                                         │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  NO EXCEPTIONS:                                                              │
│  • Нет try/catch                                                            │
│  • Errors are values (явная обработка)                                      │
│  • Explicit error propagation                                               │
│  • Predictable performance                                                  │
│                                                                              │
│  ERROR TYPE:                                                                 │
│  • type error interface { Error() string }                                  │
│  • Стандартный паттерн: if err != nil { ... }                               │
│  • Можно создавать custom error types                                       │
│  • errors.Wrap для context (pkg/errors)                                     │
│                                                                              │
│  EXAMPLE:                                                                    │
│  • file, err := os.Open("file.txt")                                         │
│  • if err != nil {                                                          │
│  •   return err                                                             │
│  • }                                                                        │
│  • defer file.Close()                                                       │
│                                                                              │
│  PANIC/RECOVER:                                                              │
│  • panic() для фатальных ошибок                                             │
│  • recover() для перехвата в defer                                          │
│  • Используется редко (только для unrecoverable errors)                     │
│  • Не заменяет normal error handling                                        │
│                                                                              │
│  ERROR WRAPPING (Go 1.13+):                                                  │
│  • return fmt.Errorf("context: %w", err)                                    │
│  • errors.Is(err, target) для проверки                                      │
│  • errors.As(err, &target) для type assertion                               │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

6. CONCURRENCY MODEL
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    GO CONCURRENCY                                            │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  GOROUTINES:                                                                 │
│  • Легковесные потоки (~2KB stack)                                          │
│  • Миллионы goroutines на одном сервере                                     │
│  • Multiplexed на OS threads                                                │
│  • Go runtime scheduler (work-stealing)                                     │
│                                                                              │
│  CHANNELS:                                                                   │
│  • CSP-style communication (communicate by sharing memory)                  │
│  • Typed channels: chan int                                                 │
│  • Buffered и unbuffered                                                    │
│  • Select для multiple channels                                             │
│  • Close для signal completion                                              │
│                                                                              │
│  SYNC PACKAGE:                                                               │
│  • WaitGroup для ожидания goroutines                                        │
│  • Mutex/RWMutex для mutual exclusion                                       │
│  • Cond для condition variables                                             │
│  • Once для single execution                                                │
│  • Atomic operations                                                        │
│                                                                              │
│  CONTEXT PACKAGE:                                                            │
│  • context.Context для cancellation                                         │
│  • Timeouts и deadlines                                                     │
│  • Value propagation через call chain                                       │
│  • Standard для HTTP, RPC, database                                         │
│                                                                              │
│  WORKER POOLS:                                                               │
│  • Легко реализовать через goroutines + channels                            │
│  • No external library needed                                               │
│  • Efficient для I/O-bound tasks                                            │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
