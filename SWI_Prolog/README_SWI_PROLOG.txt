1. УСТАНОВКА SWI-PROLOG
================================================================================

Linux (Ubuntu/Debian):
----------------------
sudo apt update
sudo apt install swi-prolog

# Проверка версии
swipl --version

# Должно вывести что-то вроде:
# SWI-Prolog version 9.0.4 for x86_64-linux

macOS (Homebrew):
-----------------
brew install swi-prolog

# Проверка
swipl --version

Windows:
--------
1. Скачать с https://www.swi-prolog.org/Download.html
2. Запустить установщик .exe
3. Добавить в PATH при установке
4. Проверка в PowerShell:
   swipl --version

Docker (универсально):
----------------------
docker run -it --rm swipl/swipl-prolog swipl

================================================================================
3. БЫСТРЫЙ СТАРТ
================================================================================

# Запуск интерпретатора
swipl

# Вы увидите:
% Welcome to SWI-Prolog (Multi-threaded, 64 bits, Version 9.0.4)
% ...
% ?-

# Первый запрос:
?- write('Hello, World!'), nl.
Hello, World!
true.

# Загрузка файла с кодом:
?- [myfile].
% или
?- consult('myfile.pl').

# Выход:
?- halt.

# Запуск скрипта из терминала:
swipl -g main -t halt myfile.pl

================================================================================
4. РЕЖИМЫ РАБОТЫ
================================================================================

Интерактивный режим (REPL):
---------------------------
swipl
?- [user].          % Ввод кода прямо в интерпретатор
|: fact(1).         % Ввод факта
|: ^D               % Ctrl+D для завершения
?- fact(X).         % Запрос
X = 1.

Скриптовый режим:
-----------------
# Файл: hello.pl
:- initialization(main).

main :-
    write('Hello from script!'), nl,
    halt.

# Запуск:
swipl hello.pl

Компиляция в байт-код (быстрее запуск):
---------------------------------------
swipl -o compiled.pl -c source.pl
swipl compiled.pl

Генерация исполняемого файла:
-----------------------------
swipl -o myprogram --stand-alone=on source.pl
./myprogram

5. ПОДВОДНЫЕ КАМНИ УСТАНОВКИ
================================================================================

⚠️ Проблема 1: Старая версия в репозиториях
   Ubuntu 20.04 может иметь версию 7.x
   Решение: Использовать PPA или скачать с сайта
   
   sudo add-apt-repository ppa:swi-prolog/stable
   sudo apt update
   sudo apt install swi-prolog

⚠️ Проблема 2: PATH не настроен (Windows)
   Решение: Добавить C:\Program Files\swipl\bin в PATH

⚠️ Проблема 3: Конфликт с GNU Prolog
   Решение: Использовать полные пути
   /usr/bin/swipl vs /usr/bin/gprolog

⚠️ Проблема 4: Кодировка файлов
   SWI-Prolog требует UTF-8
   Решение: Сохранять файлы в UTF-8 без BOM

⚠️ Проблема 5: Библиотеки XPCE (графический интерфейс)
   Могут отсутствовать в минимальной установке
   Решение: sudo apt install swi-prolog-x

6. АЛИАСЫ ДЛЯ ТЕРМИНАЛА
================================================================================

# Добавить в ~/.bashrc, ~/.zshrc, ~/.mkshrc

echo "alias pl='swipl'" >> ~/.bashrc
echo "alias pl='swipl'" >> ~/.zshrc
echo "alias pl='swipl'" >> ~/.mkshrc

# Для компиляции и запуска
echo "alias plc='swipl -g main -t halt'" >> ~/.bashrc
echo "alias plc='swipl -g main -t halt'" >> ~/.zshrc

# Применить изменения
source ~/.bashrc
source ~/.zshrc
source ~/.mkshrc 2>/dev/null || true

# Теперь:
# pl          → запуск интерпретатора
# plc file.pl → компиляция и запуск

7. ПРОВЕРКА РАБОТОСПОСОБНОСТИ
================================================================================

# Тест 1: Версия
swipl --version

# Тест 2: Простой запрос
swipl -g "write('Test OK'), nl, halt."

# Тест 3: Загрузка файла
echo "test :- write('File loaded'), nl." > test.pl
swipl -g "[test], test, halt."

# Тест 4: Проверка библиотек
swipl -g "use_module(library(lists)), halt."

# Если все тесты прошли → установка успешна!
