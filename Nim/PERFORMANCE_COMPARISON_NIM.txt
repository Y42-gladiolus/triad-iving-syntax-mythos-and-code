3. ОПТИМИЗАЦИЯ: КАК УСКОРИТЬ NIM
================================================================================

Совет 1: Используйте -d:release для production
----------------------------------------------
# МЕДЛЕННО (Debug):
nim c -r src/my_project.nim

# БЫСТРО (Release):
nim c -d:release -r src/my_project.nim

# ⚡ Ускорение: 5-10x для runtime!

Совет 2: Используйте -d:gc:arc для детерминированного GC
--------------------------------------------------------
# Стандартный GC (паузы):
nim c src/my_project.nim

# ARC (детерминированный):
nim c -d:gc:arc src/my_project.nim

# ⚡ Нет GC пауз!

Совет 3: Используйте templates для hot path
-------------------------------------------
# МЕДЛЕННО (функция):
func add(a, b: int): int =
  a + b

# БЫСТРО (template):
template add(a, b: int): untyped =
  a + b

# ⚡ Встраивается в код!

Совет 4: Избегайте аллокаций в циклах
-------------------------------------
# МЕДЛЕННО (аллокация в цикле):
for i in 0..1000:
  var s = newString(100)

# БЫСТРО (одна аллокация):
var s = newString(100)
for i in 0..1000:
  s.setLen(100)

# ⚡ Меньше аллокаций = быстрее!

Совет 5: Используйте static для compile-time вычислений
-------------------------------------------------------
# МЕДЛЕННО (runtime):
const factorial = proc(n: int): int = ...

# БЫСТРО (compile-time):
const factorial = static:
  var result = 1
  for i in 1..10: result *= i
  result

# ⚡ Вычисляется при компиляции!

-------------------

{.importc: "printf", header: "stdio.h".}
proc printf(format: cstring): cint

# Экспорт для C
proc nim_add(a, b: cint): cint {.exportc.} =
  a + b

Nim + Python (через C API):
---------------------------
# Nim компилируется в .so
# Python импортирует через ctypes

---------------------------

------------------------------
