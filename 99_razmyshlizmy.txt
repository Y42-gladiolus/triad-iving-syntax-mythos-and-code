================================================================================
ЧАСТЬ I: История Janet/mksh — просто пример
================================================================================

Сказ о том, как «Janet не вышло, mksh сработало» — не уникален.
Это один случай универсального паттерна:

    Эстетичный инструмент (Janet/.org) -> Красиво, выразительно, хрупко
    Скучный инструмент (mksh/.txt)     -> Некрасиво, ограничено, надёжно

Вы можете проверить этот паттерн прямо в этом репозитории:
- 99_fidgen-illuminati-easter-egg.org  <- красиво, сломано на GitHub
- 99_fidgen-illuminati-easter-egg.txt  <- некрасиво, работает везде

Урок не в Janet против mksh.
Урок — в **контексте исполнения**.

================================================================================
ЧАСТЬ II: «Заговор» против Скучной Правды
================================================================================

ВЕРСИЯ ЗАГОВОРА:
> «Запрещают публиковать код на Смолтолке. Замалчивают старейшин.»

СКУЧНАЯ ТЕХНИЧЕСКАЯ ПРАВДА (реальные причины):

Проблема                         | Реальная причина
---------------------------------|----------------------------------
GitHub плохо рендерит Org        | Org-mode — родной для Emacs, не для веба
#+BEGIN_SRC smalltalk не работает| 'smalltalk' нет в подсветке GitHub
:PROPERTIES: блоки скрыты        | GitHub удаляет неизвестные директивы
Бэктики (`) теряются при копировании| Терминал интерпретирует спецсимволы
Кавычки «ёлочки» становятся прямыми| Конвертация кодировки при вставке
Отступы «плывут»                 | Обработка пробелов различается в редакторах

РЕШЕНИЕ (немодное, но работающее):
> Сохранить как .txt — чистый текст, без попыток рендеринга.
> GitHub покажет как есть. Отступы сохранятся. Без магии.

================================================================================
ЧАСТЬ III: Почему это больно (эстетически)
================================================================================

.org с директивами Org-mode:
    - Ощущается как запись священных табличек
    - В Emacs: рай. На GitHub: сломано.

.txt как чистый текст:
    - Ощущается как список покупок
    - В Emacs: просто. На GitHub: работает.

Это не эстетично. Это не модно.
Но почему-то — надёжнее.
