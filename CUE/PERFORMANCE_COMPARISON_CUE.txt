3. ОПТИМИЗАЦИЯ: КАК УСКОРИТЬ CUE
================================================================================

Совет 1: Используйте definitions для reusable schemas
-----------------------------------------------------
# МЕДЛЕННО (duplicate schemas):
person1: {name: string, age: int}
person2: {name: string, age: int}

# БЫСТРО (definitions):
#Person :: {name: string, age: int}
person1: #Person
person2: #Person

# ⚡ Definitions reusable и быстрее!

Совет 2: Используйте import для standard library
------------------------------------------------
# МЕДЛЕННО (manual validation):
validEmail: email =~ "^[^@]+@[^@]+$"

# БЫСТРО (regexp package):
import "regexp"
validEmail: regexp.match("^[^@]+@[^@]+$", email)

# ⚡ Standard library оптимизирована!

Совет 3: Используйте multi-file organization
--------------------------------------------
# МЕДЛЕННО (one big file):
// 1000 lines in one file

# БЫСТРО (multi-file):
// schema.cue (200 lines)
// config.cue (300 lines)
// env-dev.cue (200 lines)
// env-prod.cue (300 lines)

# ⚡ Multi-file быстрее для evaluation!

Совет 4: Используйте cue vet для validation
-------------------------------------------
# МЕДЛЕННО (manual checking):
// Check each field manually

# БЫСТРО (cue vet):
cue vet config.cue

# ⚡ cue vet оптимизирован для validation!

Совет 5: Используйте export для production
------------------------------------------
# МЕДЛЕННО (manual export):
// Convert CUE to JSON manually

# БЫСТРО (cue export):
cue export config.cue --out json

# ⚡ cue export быстрый и надежный!

8. ИНТЕГРАЦИЯ CUE С ДРУГИМИ ЯЗЫКАМИ
================================================================================

CUE + Go (native integration):
------------------------------
// Go code импортирует CUE
import "cuelang.org/go/cue"

// CUE генерирует Go code
cue gen config.cue

CUE + Kubernetes:
-----------------
# Validate Kubernetes manifests
cue vet k8s-manifest.cue

# Export to YAML
cue export k8s-manifest.cue --out yaml | kubectl apply -f -

CUE + Terraform:
----------------
# Validate Terraform configs
cue vet terraform.cue

# Export to JSON
cue export terraform.cue --out json

CUE + CI/CD:
------------
# GitHub Actions
- name: Validate CUE
  run: cue vet ./...

# GitLab CI
validate:
  script:
    - cue vet ./...

9. БЕНЧМАРК: ПРОВЕРЬТЕ САМИ
================================================================================

Создайте benchmark.cue:
-----------------------
#LargeConfig :: {
  items: [...{
    name: string
    value: int
    active: bool
  }]
}

config: #LargeConfig & {
  items: [{
    name: "item1"
    value: 100
    active: true
  }, ...] // 1000 items
}

Запуск:
-------
time cue eval benchmark.cue
time cue vet benchmark.cue

Сравнение с YAML:
-----------------
# CUE будет быстрее для validation!
# YAML требует external validation
