3. ОПТИМИЗАЦИЯ: КАК УСКОРИТЬ COQ
================================================================================

Совет 1: Используйте .vo кэш
----------------------------
# Первый build (медленно):
coqc file.v

# Последующие (быстро, использует .vo):
Require Import file.

Совет 2: Используйте тактики эффективно
---------------------------------------
(* МЕДЛЕННО (ручное доказательство): *)
Theorem slow : forall n m : nat, n + m = m + n.
Proof.
  intros n m.
  induction n.
  - simpl. rewrite Nat.add_0_r. reflexivity.
  - simpl. rewrite IHn. rewrite Nat.add_succ_r. reflexivity.
Qed.

(* БЫСТРО (автоматика): *)
Theorem fast : forall n m : nat, n + m = m + n.
Proof.
  intros.
  rewrite Nat.add_comm.
  reflexivity.
Qed.

Совет 3: Используйте Ltac для автоматизации
-------------------------------------------
Ltac auto_arith :=
  try omega;
  try ring;
  try lia.

Theorem ltac_example : forall n m : nat, n < m -> n + 1 <= m.
Proof.
  intros.
  auto_arith.  (* Автоматически *)
Qed.

9. БЕНЧМАРК: ПРОВЕРЬТЕ САМИ
================================================================================

Создайте Benchmark.v:
---------------------
Require Import Arith.

Fixpoint sumTo (n : nat) : nat :=
  match n with
  | 0 => 0
  | S n' => n + sumTo n'
  end.

Theorem sumTo_formula : forall n : nat,
  2 * sumTo n = n * (n + 1).
Proof.
  intros n.
  induction n as [| n' IH].
  - simpl. ring.
  - simpl. rewrite IH. ring.
Qed.

Compute (sumTo 1000).

Компиляция и запуск:
--------------------
coqc Benchmark.v
coqtop -l Benchmark.v

Сравнение с экстракцией:
------------------------
Require Import Extraction.
Extraction "benchmark.ml" sumTo.
ocaml benchmark.ml  (* Быстрый OCaml код! *)
