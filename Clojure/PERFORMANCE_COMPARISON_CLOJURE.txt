3. ОПТИМИЗАЦИЯ: КАК УСКОРИТЬ CLOJURE
================================================================================

Совет 1: Используйте transducers
--------------------------------
; МЕДЛЕННО (chain):
(->> data
     (filter pred)
     (map transform)
     (reduce +))

; БЫСТРО (transducer):
(transduce (comp (filter pred) (map transform)) + 0 data)

; ⚡ Transducers эффективнее!

Совет 2: Используйте type hints
-------------------------------
; МЕДЛЕННО (без hints):
(defn sum [nums]
  (reduce + 0 nums))

; БЫСТРО (с hints):
(defn ^long sum [^java.util.List nums]
  (reduce + 0 nums))

; ⚡ Type hints предотвращают reflection!

Совет 3: Используйте primitives
-------------------------------
; МЕДЛЕННО (boxed):
(defn sum [nums]
  (reduce + 0 nums))

; БЫСТРО (primitives):
(defn sum [nums]
  (reduce + 0^long nums))

; ⚡ Primitives быстрее boxed!

Совет 4: Используйте AOT компиляцию
-----------------------------------
; МЕДЛЕННО (interpretation):
clojure -M -m my-project.core

; БЫСТРО (AOT):
lein compile
java -jar my-project.jar

; ⚡ AOT ускоряет startup!

Совет 5: Используйте memoize
----------------------------
; МЕДЛЕННО (без кэша):
(defn factorial [n]
  (if (<= n 1) 1 (* n (factorial (dec n)))))

; БЫСТРО (с кэшем):
(def factorial (memoize
  (fn [n]
    (if (<= n 1) 1 (* n (factorial (dec n)))))))

; ⚡ Memoize кэширует результаты!

9. БЕНЧМАРК: ПРОВЕРЬТЕ САМИ
================================================================================

Создайте benchmark.clj:
-----------------------
(ns benchmark)

(defn sum-enum [n]
  (reduce + 0 (range n)))

(defn -main []
  (let [start (System/currentTimeMillis)
        result (sum-enum 1000000000)
        end (System/currentTimeMillis)]
    (println "Sum:" result)
    (println "Time:" (- end start) "ms")))

Запуск:
-------
lein run
clojure -M -m benchmark

Сравнение с Java:
-----------------
# Clojure будет в 2-3 раза медленнее Java
# Но с лучшей выразительностью!
