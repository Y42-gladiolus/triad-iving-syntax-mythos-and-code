1. АРХИТЕКТУРА И ДИЗАЙН
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    HASKELL ARCHITECTURE                                      │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  IMPLEMENTATION:                                                             │
│  • Написан на C и Haskell (GHC - Glasgow Haskell Compiler)                  │
│  • Компилируется в native code                                              │
│  • LLVM backend (опционально)                                               │
│  • Кроссплатформенный (Linux, macOS, Windows)                               │
│                                                                              │
│  DESIGN PHILOSOPHY:                                                          │
│  • Pure functional programming (нет side effects)                           │
│  • Lazy evaluation (вычисления по требованию)                               │
│  • Strong static typing (с type inference)                                  │
│  • Type classes (ad-hoc polymorphism)                                       │
│  • Immutability (все данные неизменяемы)                                    │
│  • No null (Maybe для опциональных значений)                                │
│                                                                              │
│  KEY FEATURES:                                                               │
│  • Pure functions (нет side effects)                                        │
│  • Lazy evaluation (ленивые вычисления)                                     │
│  • Type inference (не нужно annotating всё)                                 │
│  • Pattern matching (мощный и выразительный)                                │
│  • Type classes (как interfaces но мощнее)                                  │
│  • Monads (для side effects)                                                │
│  • Algebraic data types                                                     │
│  • Higher-kinded types                                                      │
│                                                                              │
│  STATUS (2024):                                                              │
│  • Stable release (GHC 9.x)                                                 │
│  • Production ready                                                         │
│  • Mature ecosystem (1990+)                                                 │
│  • Open source (BSD license)                                                │
│  • Created by Haskell Committee                                             │
│  • 30+ лет развития                                                         │
│                                                                              │
│  COMPARISON:                                                                 │
│  • vs OCaml: Haskell ленивее, строже типы                                   │
│  • vs Scala: Haskell pure, Scala hybrid                                     │
│  • vs F#: Haskell ленивее, мощнее type system                               │
│  • vs Clojure: Haskell static, Clojure dynamic                              │
│  • vs Erlang: Haskell pure, Erlang actor model                              │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

5. ERROR HANDLING
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    HASKELL ERROR HANDLING                                    │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  NO EXCEPTIONS (по умолчанию):                                               │
│  • Нет try/catch для pure code                                              │
│  • Either для error handling                                                │
│  • Maybe для optional values                                                │
│  • Type-safe error handling                                                 │
│                                                                              │
│  MAYBE:                                                                      │
│  • data Maybe a = Nothing | Just a                                          │
│  • Для optional values                                                      │
│  • Нет null pointer exceptions                                              │
│  • Safe by design                                                           │
│                                                                              │
│  EITHER:                                                                     │
│  • data Either a b = Left a | Right b                                       │
│  • Left для ошибок, Right для успеха                                        │
│  • Type-safe error propagation                                              │
│  • Как Result в Rust                                                        │
│                                                                              │
│  EXCEPTIONS (для IO):                                                        │
│  • try/catch для IO monad                                                   │
│  • Exception hierarchy                                                      │
│  • Custom exceptions                                                        │
│  • Как Java, Python                                                         │
│                                                                              │
│  EXAMPLE:                                                                    │
│  • safeDivide :: Int -> Int -> Maybe Int                                    │
│  • safeDivide _ 0 = Nothing                                                 │
│  • safeDivide x y = Just (x `div` y)                                        │
│                                                                              │
│  MONAD TRANSFORMERS:                                                         │
│  • ErrorT для error handling                                                │
│  • ExceptT для exceptions                                                   │
│  • MonadError type class                                                    │
│  • Composable error handling                                                │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

6. LAZY EVALUATION
================================================================================

┌──────────────────────────────────────────────────────────────────────────────┐
│                    HASKELL LAZY EVALUATION                                   │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  WHAT IS LAZINESS:                                                           │
│  • Вычисления по требованию                                                 │
│  • Нет вычислений пока результат не нужен                                   │
│  • Infinite data structures возможны                                        │
│  • Composability и modularity                                               │
│                                                                              │
│  BENEFITS:                                                                   │
│  • Infinite lists: [1..]                                                    │
│  • Short-circuit evaluation                                                 │
│  • Better modularity                                                        │
│  • Separation of concerns                                                   │
│  • Efficient pipelines                                                      │
│                                                                              │
│  EXAMPLES:                                                                   │
│  • take 5 [1..]  # [1,2,3,4,5] (не вычисляет весь список)                   │
│  • head (filter even [1..10])  # 2 (не фильтрует всё)                       │
│  • fibonacci = 0 : 1 : zipWith (+) fibonacci (tail fibonacci)               │
│                                                                              │
│  SPACE LEAKS:                                                                │
│  • Thunks накапливаются в памяти                                            │
│  • Нужно использовать strictness когда нужно                                │
│  • Bang patterns (!x)                                                       │
│  • seq для forced evaluation                                                │
│  • Profile для detection                                                    │
│                                                                              │
│  STRICTNESS CONTROL:                                                         │
│  • Bang patterns: f !x = x + 1                                              │
│  • seq: x `seq` y (вычисляет x перед y)                                     │
│  • Control.DeepSeq: force, deepseq                                          │
│  • Pragmas: {-# STRICT #-}                                                  │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
